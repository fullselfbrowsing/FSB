# 23-02 Plan: Static Dashboard Renderer Migration

## Goal

Route the static dashboard's snapshot rendering and mutation application through the shared PhantomStream viewer wrapper while preserving FSB preview state, side-channel UI, diagnostics, and remote-control affordances.

## Tasks

1. Add a viewer host container to `showcase/dashboard.html` and keep existing FSB chrome/overlays above it.
2. Replace manual iframe `srcdoc` construction in `showcase/js/dashboard.js` with wrapper viewer dispatch for `ext:dom-snapshot`.
3. Replace manual `[data-fsb-nid]` mutation application with wrapper viewer dispatch for `ext:dom-mutations`.
4. Keep existing `shouldAcceptPreviewMessage`, stale-session checks, stale-mutation diagnostics/resync, loading/restricted/error/frozen/paused states, URL bar, and task/remote-control state outside the wrapper.
5. Forward scroll/overlay/dialog payloads into the viewer while preserving existing host-side progress badge, frozen identity, dialog state, and tooltip counters.
6. Preserve remote-control overlay geometry by deriving current mapping from the viewer wrapper rather than stale iframe scale assumptions.
7. Update static dashboard tests so they assert package-backed renderer usage and absence of legacy generic snapshot/diff implementation.

## Done When

- Static dashboard snapshot and mutation paths dispatch through the shared PhantomStream viewer wrapper.
- Static dashboard no longer owns generic iframe HTML assembly or generic nid-addressed diff application.
- Existing preview states and side-channel UI continue to pass source-contract tests.
