# 23-04 Plan: Viewer Closeout And Diagnostics

## Goal

Close Phase 23 by proving renderer-backed dashboards preserve side-channel UI, frozen states, diagnostics, resync behavior, and layout/remote-control affordances.

## Tasks

1. Add closeout tests for preview state compatibility:
   - loading;
   - streaming;
   - paused;
   - disconnected;
   - restricted;
   - frozen-disconnect;
   - frozen-complete;
   - error.
2. Verify side-channel rendering remains intact for scroll, action glow, progress/client badge, native dialogs, and final/frozen overlay state.
3. Verify diagnostic counters still expose last frame, last snapshot, mutation/apply failure counts, stale counts, and viewer health/state without mirrored content leakage.
4. Verify resync still sends the existing dashboard stream-start message and remains latched until a new snapshot.
5. Run static/Angular parity tests plus extension validation.
6. Create Phase 23 validation and summary artifacts; mark VIEW-01..VIEW-04 complete only if both dashboards are renderer-backed.
