# 22-03 Plan: Side Channels And Watchdogs

## Goal

Preserve FSB overlay, dialog, diagnostics, and watchdog behavior through the PhantomStream-backed adapter.

## Tasks

1. Implement the FSB overlay provider passed to `createCapture`, preserving glow/progress/client/session/result fields.
2. Ensure `domStreamRequestOverlay` forces a package-backed overlay broadcast.
3. Preserve dialog event forwarding shape and rate-limited diagnostics.
4. Preserve content self-watchdog and SW-side stale-flush diagnostics through package payloads and `FSB.domStream.getStaleFlushCount()`.
5. Update capture-side tests to assert behavior through adapter-visible messages rather than source-body implementation details.
