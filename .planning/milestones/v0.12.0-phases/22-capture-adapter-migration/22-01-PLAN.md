# 22-01 Plan: Capture Adapter Seam

## Goal

Make PhantomStream capture importable from FSB's classic content-script runtime and preserve the current control-message surface before replacing snapshot/diff internals.

## Tasks

1. Add an esbuild browser IIFE entry that imports `createCapture` and protocol constants from `@full-self-browsing/phantom-stream` and exposes a stable global bridge for `content/dom-stream.js`.
2. Generate and commit the content-script bundle under `extension/dist/content/`.
3. Update `extension/background.js` `CONTENT_SCRIPT_FILES` to inject the bundle before `content/dom-stream.js`, and include `content/dom-stream.js` in the primary injection list.
4. Add a focused test that asserts the bundle file exists, the injection order is correct, and `dom-stream.js` references the package-backed bridge rather than owning capture construction directly.

## Verification

```bash
npm run build
node tests/phantom-stream-content-bundle.test.js
node tests/dashboard-stream-readiness-ping.test.js
node tests/dashboard-stream-pending-intent.test.js
```

## Done When

- `dom-stream.js` can access PhantomStream capture in a classic content-script context.
- The ready/probe and reinjection paths still include the stream module.
- The adapter seam is test-pinned before behavior replacement.
