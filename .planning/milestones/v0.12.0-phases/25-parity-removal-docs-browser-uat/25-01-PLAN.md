# 25-01 Plan: Stream Engine Removal

## Goal

Remove FSB-owned duplicate stream-engine residue now that capture, renderer, protocol, relay, and remote-control paths are package-backed or explicit adapters.

## Tasks

1. Remove the temporary legacy `data-fsb-nid` stamping bridge from `extension/content/dom-stream.js`.
2. Preserve FSB-specific adapter behavior: background action names, stream identity fields, stale flush diagnostics, overlay exclusion, dialog/scroll/overlay forwarding, and resume-as-fresh-snapshot behavior.
3. Update capture/perf tests so they assert package-backed adapter boundaries and the absence of old local serializer/stamping code.
4. Add focused source-contract coverage if needed for "no duplicate capture/renderer/relay engine" boundaries.
5. Record remaining adapter code that is intentionally FSB-specific, not duplicate PhantomStream ownership.

- Production stream adapters no longer contain legacy local capture/renderer engine code or temporary `data-fsb-nid` stamping.
- Tests distinguish package-owned engines from intentional FSB adapters.
- PARITY-03 has automated evidence.
