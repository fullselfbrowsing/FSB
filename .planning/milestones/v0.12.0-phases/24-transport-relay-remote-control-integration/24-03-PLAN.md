# 24-03 Plan: Recovery Parity

## Goal

Prove the package-backed transport/relay seams preserve stream recovery across dashboard reconnect, extension reconnect, service-worker wake, content late readiness, and watchdog snapshot request paths.

## Tasks

1. Verify dashboard reconnect still receives extension snapshot, page-ready, stream-state, and task recovery state.
2. Verify extension reconnect still preserves stream intent and re-emits recovery state.
3. Verify service-worker wake keeps `_pendingStreamStart` and stream-state recovery behavior intact.
4. Verify late content readiness still re-arms parked `dash:dom-stream-start` once `domStreamReady` arrives.
5. Verify watchdog/request-snapshot paths continue using existing FSB stream messages and package-compatible constants.
