# 24-02 Plan: Relay Compatibility

## Goal

Adopt PhantomStream relay primitives where they fit, or document and test an explicit FSB compatibility adapter for `showcase/server/src/ws/handler.js`.

## Tasks

1. Compare FSB relay behavior to `@full-self-browsing/phantom-stream/relay` exports:
   - hash-key rooms;
   - extension/dashboard roles;
   - frame classification;
   - 1 MiB message cap;
   - 16 MiB backpressure drop behavior;
   - diagnostics and disconnect behavior.
2. Integrate package relay helpers only if they preserve FSB pairing and task/status traffic.
3. If full adoption is unsafe, keep the FSB relay and add a documented compatibility adapter around package constants/classifiers.
4. Extend relay tests to prove stream frames, task frames, compression envelopes, oversized frames, and backpressure behavior still match current production semantics.

## Verification

```bash
node tests/server-ws-backpressure.test.js
node tests/phantom-stream-protocol-envelope.test.js
npm run validate:extension
git diff --check
```

## Done When

- RELAY-02 is proven by tests and documented as package-backed or intentionally compatibility-backed.
- RELAY-03 relay-side compression classification remains self-identifying and stateless.
