<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| persisted trigger snapshot -> Chrome tabs API | Snapshot tab and agent ownership data authorize a reload side effect. |
| Chrome tab reload -> content script read | After reload, untrusted page state is read through the content script and staged for SW evaluation. |
| content read response -> lifecycle seam | A typed value or attention outcome crosses into persisted trigger state and fire evaluation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01 | Tampering / Denial of Service | refresh-poll pre-reload ownership gate | mitigate | `fsbTriggerValidateRefreshPollOwnership` uses `hasAgent`, `getOwner`, and `isOwnedBy` before any reload. `TAB_NOT_OWNED` returns before `chrome.tabs.reload`; tests assert ownership code appears before reload. |
| T-17-04 | Tampering | stale selector / missing element treated as empty text | mitigate | Consume Plan 17-02 `ELEMENT_NOT_FOUND` and write `status:'needs_attention'` with `attention_reason:'element_not_found'` without calling `handleTriggerAlarm`. |
| T-17-03 | Denial of Service | reload hammering after non-fire | mitigate | Reuse Plan 17-01 `scheduleNextRefreshPollAlarm` after non-terminal ticks; no standing timers or immediate loops. |
| T-17-02 | Spoofing / Tampering | challenge/login/CAPTCHA page false fire | transfer | This plan handles missing elements; Plan 17-04 adds blocked-page classification before staging values. |
</threat_model>
