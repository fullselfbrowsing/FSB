## Phase-Specific UI Contract

| Area | Contract |
|------|----------|
| Placement | Add one `settings-card` immediately after the existing Agent Concurrency card inside `advanced-settings-grid`, so it occupies the adjacent grid cell on desktop and stacks naturally on mobile. Do not make it `full-width`. |
| Header | Card title: `Trigger Concurrency`. Subtitle: `Maximum simultaneous trigger watches`. Header icon: `fas fa-bolt` inside the existing `settings-card-icon` treatment. |
| Control shape | Clone the Agent Concurrency shape: label row, numeric value badge, native number input, reset button, range helper, validation hint, and active counter. Do not add a slider, chart, modal, new section, or decorative redesign. |
| Element IDs | Use `fsbTriggerCap`, `fsbTriggerCapDisplay`, `fsbTriggerCapReset`, `fsbTriggerCapValidation`, and `fsbTriggerCapCurrentActive`. |
| Storage | Read/write `chrome.storage.local.fsbTriggerCap`. Default `8`, minimum `1`, maximum `64`, step `1`. Clamp on input, load, and save. |
| Reset interaction | `Reset to default (8)` sets the input and value badge to `8`, hides validation, refreshes the active counter, and marks settings unsaved. |
| Validation | Invalid raw input shows `Must be between 1 and 64`, then the control normalizes to an integer in `[1, 64]`. Saving persists the clamped value. |
| Active counter | Render `N of M active`. Count `chrome.storage.session.fsbTriggerRegistry.records` where status is `armed`, `needs_attention`, or `blocked`. Exclude terminal `fired`, `timed_out`, and `stopped`; exclude malformed records. |
| Counter updates | Refresh on initial load, on input/reset, and through `chrome.storage.onChanged` for `session/fsbTriggerRegistry` and `local/fsbTriggerCap`, debounced at `100ms` to match the agent counter pattern. |
| Failure behavior | Counter reads are informational and best-effort. Storage or runtime errors must not throw into the options page and must not show a toast. |
| Scope limits | No shadcn, no third-party registry blocks, no new visual system, no marketing/landing patterns, and no browser-driven UI harness for this card. |

Required card copy:

| Element | Copy |
|---------|------|
| Title | Trigger Concurrency |
| Subtitle | Maximum simultaneous trigger watches |
| Label | Concurrency Cap |
| Reset button | Reset to default (8) |
| Helper | Default 8. Range 1 to 64. Lower = predictable resource use; higher = more parallel watches. Lowering this value while triggers are active does not stop existing watches; new triggers past the new cap are rejected. |
| Validation | Must be between 1 and 64 |
| Counter | 0 of 8 active |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Validation and counter top offsets; tight inline gaps |
| sm | 8px | Reset button top margin; label-to-control spacing |
| md | 16px | Default input padding and compact element spacing |
| lg | 24px | Card header/content padding and grid gaps |
| xl | 32px | Section-level spacing inherited from Advanced Settings |
| 2xl | 48px | Major section breaks inherited from the dashboard shell |
| 3xl | 64px | Page-level spacing only; not used inside the card |

Exceptions: none for new UI. The Trigger Concurrency card must use inherited existing classes; literal margins are allowed only when cloning Agent Concurrency (`8px` reset top margin, `4px` validation/counter top margin).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#fffdfb` / `var(--bg-secondary)` | Dashboard background and Advanced Settings section surface |
| Secondary (30%) | `#ffffff` / `var(--bg-primary)` | Settings cards, card headers, and number input surface |
| Accent (10%) | `#ff6b35` / `var(--primary-color)` | Trigger card icon, numeric value badge, input focus border/ring, and existing accent affordances only |
| Destructive | `#dc2626` / `var(--error-color)` | Destructive actions only; no destructive action is added in this phase |

Accent reserved for: Trigger Concurrency card icon, `fsbTriggerCapDisplay` value badge, focused `fsbTriggerCap` input, and existing theme focus rings. Do not use accent for helper text, active counter text, reset button background, or every interactive element.

Validation text should visually match the Agent Concurrency validation hint (`Must be between 1 and 64`) and remain a small inline error hint, not a destructive action.

## Copywriting Contract

| Element | Copy |
|---------|------|

Tone: terse operational settings copy. Avoid explanatory product marketing, promises of push delivery, or language implying server-side monitoring.
