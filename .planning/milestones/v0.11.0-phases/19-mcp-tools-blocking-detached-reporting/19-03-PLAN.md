  <behavior>
    - Existing fire-once edge behavior remains unchanged when no hysteresis is supplied.
    - For threshold `>` / `>=`, after a satisfied state, `was_satisfied` remains true until current value is below `target - hysteresis`.
    - For threshold `<` / `<=`, after a satisfied state, `was_satisfied` remains true until current value is above `target + hysteresis`.
    - For `percent_change`, after a satisfied state, `was_satisfied` remains true until absolute percent change is below `percent - hysteresis` with a floor of `0`.
    - Regex/contains/equals keep existing boolean edge behavior unless a clear, tested hysteresis rule already exists.
  </behavior>
  <action>
    In `extension/utils/trigger-manager.js`, keep `evaluate()` pure and add helper logic such as `applyHysteresis(condition, safeSnap, result, opts)` after `evaluateOne/evaluateCompound` returns and before `isEdge` is computed.

    Implement exact rules:
    - `const hysteresis = Number(condition.hysteresis); const margin = Number.isFinite(hysteresis) && hysteresis > 0 ? hysteresis : 0;`
    - if `safeSnap.was_satisfied !== true` or `margin === 0`, preserve existing `satisfiedNow = result.satisfied === true`;
    - for threshold numeric conditions with operator `>` or `>=`, parse current raw value and target using the existing extractor. If parse succeeds and current value is greater than or equal to `target - margin`, force `satisfiedNow = true`; only values below `target - margin` reset false;
    - for threshold numeric conditions with operator `<` or `<=`, force `satisfiedNow = true` until current value is greater than `target + margin`;
    - for `percent_change`, compute absolute percent change from baseline using existing extractor parse helpers and force `satisfiedNow = true` while `pct >= Math.max(0, Math.abs(condition.percent) - margin)`;
    - on parse errors, keep existing parse_error behavior and do not advance/reset edge state.

    Add tests in `tests/trigger-manager.test.js`:
    - threshold greater-than with `baseline:'90'`, target `100`, hysteresis `5`: first value `101` fires; next value `103` with `was_satisfied:true` no-fires and remains satisfied; value `96` remains satisfied; value `94` resets `was_satisfied:false`; value `101` fires again;
    - threshold less-than symmetric case;
    - percent_change case with baseline `100`, percent `10`, hysteresis `2`: `112` fires, `111` no-fires remains satisfied, `107` resets, `112` fires again;
    - existing no-hysteresis edge tests still pass.
  </action>

## Trust Boundaries

| Boundary | Description |

| Agent registry -> trigger lifecycle | Owner release crosses from agent ownership management into trigger cleanup. |
| Detached trigger lifetime -> cap enforcement | Long-lived triggers must not outlive owner/tab/TTL and consume cap slots forever. |
