<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Watch layer -> lifecycle event | Page-derived values and URLs enter persisted trigger events. |
| Lifecycle snapshot -> MCP status/list | Stored events may include page data and must remain owner-scoped. |
| Blocking timeout -> cleanup helpers | Timeout path mutates observer/pulse/alarm/storage state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-04 | Information Disclosure | fire event status/list projection | mitigate | Apply existing ownership filtering before returning `last_event`; tests assert denied snapshots do not return event data. |
| T-19-05 | Tampering / Integrity | terminal event write-back | mitigate | Write `last_event` in the same storage transition as `status:'fired'` or `status:'timed_out'`; tests fail on missing event/status fields. |
| T-19-06 | Denial of Service | timeout cleanup | mitigate | Timeout clears observer/pulse/watchdog/alarm and changes status away from `armed`, freeing cap slots. |
</threat_model>
