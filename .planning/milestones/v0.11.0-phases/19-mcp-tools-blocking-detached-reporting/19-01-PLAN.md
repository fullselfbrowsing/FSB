<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP caller -> shared trigger schema | Untrusted tool inputs add reporting fields and optional caller-supplied `trigger_id`. |
| MCP server -> extension bridge | Long-running request can hold transport resources and receive progress. |
| Extension bridge -> trigger store | Blocking wait reads persisted trigger snapshots that may contain page data. |
| Bridge reconnect -> status lookup | Recovery must not leak another agent's trigger status. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-01 | Denial of Service | blocking `trigger()` bridge request | mitigate | Enforce `240_000ms` max safety ceiling, paired timer cleanup, and detached auto-convert. |
| T-19-02 | Information Disclosure | bridge-disconnect status recovery | mitigate | Lookup by known `trigger_id` through existing agent-scoped bridge/status route; do not read raw store server-side. |
| T-19-03 | Spoofing / Tampering | caller-supplied `trigger_id` and reporting params | mitigate | Treat id as correlation only; background ownership and condition validation remain authoritative; schema tests lock additive fields. |
</threat_model>
