<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP/autopilot caller -> background trigger handlers | Untrusted tool parameters cross into trigger arm/stop/status/list orchestration. |
| content triggerRead -> persisted trigger snapshot | Page DOM values become baseline/reported snapshot values used by the fire-condition engine. |
| trigger snapshot -> companion response | Persisted owner, status, and attention details are returned to a caller and must be access-controlled. |
| stop request -> content/lifecycle cleanup | A cancellation request crosses into content observer/pulse cleanup and lifecycle alarm/snapshot deletion. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-01 | Denial of Service | `fsbTriggerDispatchToolRequest` / stop handler | mitigate | Stop is a bounded background handler with no watcher ownership in MCP; Plan 18-03/04 put it on bypass/direct paths. |
| T-18-02 | Information Disclosure / Elevation of Privilege | status/list/stop handlers | mitigate | Compare persisted `agent_id` and `ownership_token` with caller context before returning or mutating snapshots. |
| T-18-03 | Tampering | arm condition validation | mitigate | Validate condition kind/required fields before `FsbTriggerManager.armTrigger`; malformed input returns `TRIGGER_CONDITION_INVALID`. |
| T-18-04 | Denial of Service | stop cleanup | mitigate | Stop calls observer stop, pulse stop, watchdog clear, then lifecycle clear; source-order tests lock cleanup. |
| T-18-05 | Information Integrity | status/list projection | mitigate | Status/list read `FsbTriggerStore` snapshots and never synthesize from SW heap or alarm names. |
</threat_model>
