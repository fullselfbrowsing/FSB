<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client -> MCP registrar | Client-supplied trigger parameters enter TypeScript registration and bridge payload construction. |
| MCP registrar -> extension bridge | Agent identity and ownership token are attached to trigger messages crossing into the extension. |
| MCP TaskQueue -> cancellation/status companions | Companion scheduling must bypass mutation serialization even if another mutation never resolves. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-01 | Denial of Service | `registerTriggerTools`, `TaskQueue` | mitigate | Direct companion dispatch plus actual TaskQueue bypass test while a mutation is pending. |
| T-18-02 | Information Disclosure / Elevation of Privilege | agent-scoped bridge payload | mitigate | Use `sendAgentScopedBridgeMessage` so background receives FSB-minted `agentId` and ownership token, not caller-authored identity. |
| T-18-03 | Tampering | MCP schema conversion | mitigate | Source schemas only from shared `TOOL_REGISTRY`; background validation in Plan 18-02 remains the enforcement point for nested conditions. |
| T-18-06 | Information Integrity | MCP registration surface | mitigate | Smoke tests assert MCP registered handlers come from `registerTriggerTools`, not duplicated manual schemas. |
</threat_model>
