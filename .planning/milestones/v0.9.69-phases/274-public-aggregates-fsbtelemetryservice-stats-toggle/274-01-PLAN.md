    **Step 3 — create `showcase/server/src/routes/public-stats.js`** (NEW, per CONTEXT decisions §Public endpoint mount + auth + Cache strategy):

    Module exports a factory `createPublicStatsRouter(db, queries)` that returns an Express router. The router has NO auth middleware — anonymous-by-design (per STATS-04 and CONTEXT decisions).

    Module-scoped in-process memo `const memo = new Map()` keyed by route path (`'/global'`, `'/global/series'`). Entry shape: `{ body: string, etag: string, expiresAt: number }`. TTL = 30 seconds. LRU eviction at 100 entries (only 2 routes, so eviction will almost never fire — but include the cap so a misconfigured caller cannot blow memory).

    ETag computation: `crypto.createHash('sha256').update(body).digest('hex').slice(0, 16)` wrapped in double quotes per RFC 7232 (i.e. `"abc123def4567890"`).

    Helper `buildHeadlineJson(db, queries)`:
    1. Read raw rows from `queries.getPublicHeadlineRows()`.
    2. Get live counts from active-tracker: `active_users_now = countActiveUsers(5*60*1000)`; `active_agents_now = getActiveAgentSum(10*60*1000)`.
    3. Parse `latest_global.popular_mcp_json` and `popular_agent_json` defensively (try/catch, default to `[]`). Map raw rows `{mcp_client, uniq}` → `{label, uniq}` and `{agent, uniq}` → `{label, uniq}` (rename for client-friendly shape — see FSBTelemetryHeadline contract in <interfaces>).
    4. Compute `avg_agents_per_user = active_users_now > 0 ? Math.round((active_agents_now / active_users_now) * 10) / 10 : 0`.
    5. Compute `active_agents_bucket = bucketAgents(active_agents_now)` (import from active-tracker.js).
    6. Return the FSBTelemetryHeadline JSON object.

    Helper `buildSeriesJson(queries)`:
    1. Read raw rows from `queries.getPublicSeriesRows()`.
    2. Map each window's rows: `{ day_utc, unique_installs, tokens: tokens_in_sum + tokens_out_sum, agents_active: agents_active_sum }`.
    3. Return FSBTelemetrySeries shape `{ d30, d90, d365 }`.

    Memo-wrapping middleware (factored into a `memoize(routeKey, builder)` helper):
    1. Check memo[routeKey]; if entry exists AND `entry.expiresAt > Date.now()`, treat as cache hit.
    2. On cache miss, call `builder()`, JSON-stringify the result, compute ETag, store `{body, etag, expiresAt: Date.now() + 30000}`.
    3. Compare request's `If-None-Match` header with stored ETag. If equal → respond `304` with NO body (set ETag + Cache-Control headers only).
    4. Otherwise respond `200` with `Content-Type: application/json`, `Cache-Control: max-age=60`, `ETag: <stored>`, and the body.
    5. Apply `res.removeHeader('Set-Cookie')` defensively before send (the cors middleware up the stack has `credentials: true` but on a path with NO auth it should never set cookies — this is belt-and-suspenders to satisfy `tests/server-public-stats-no-auth.test.js`).

    Routes:
    - `GET /global` → `memoize('/global', () => buildHeadlineJson(db, queries))`
    - `GET /global/series` → `memoize('/global/series', () => buildSeriesJson(queries))`

    Defensive normalization on shape: every numeric field guaranteed to be a JS `number` (not BigInt — better-sqlite3 returns numbers by default for INTEGER columns ≤ 2^53; the SUM() columns can theoretically exceed this for tokens, but at current scale it's safe; we accept this until a billion-tokens-test fails).

    Export: `module.exports = createPublicStatsRouter; module.exports.createPublicStatsRouter = createPublicStatsRouter;` (parallel to the existing telemetry router export style).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet → public-stats router | Anonymous GET requests with NO auth. Untrusted client; treat all headers (`If-None-Match`) as adversarial input. |
| extension client → /api/telemetry/events → active-tracker | Already gated by Phase 273 validation + rate-limit + per-UUID budget. Active-tracker only sees post-validation data. |
| public-stats route → SQLite | Read-only access to rollup + global-aggregates tables. No untrusted data flows into SQL — query parameters are constant (date ranges only). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-274-01 | Information Disclosure | public-stats `/global` response body | mitigate | Strict shape allowlist: response is hand-built from typed fields (FSBTelemetryHeadline interface). No `SELECT *`. Test `tests/server-public-stats-no-auth.test.js` regex-asserts no UUIDs, no `ip_hash`, no `event_id`, no `model` strings leak. The popular_mcp_clients list reads from housekeeper-pre-filtered JSON (k>=5 floor already applied per D-07). |
| T-274-02 | Information Disclosure | active_users_now / active_agents_now | accept | Counts/sums only escape the in-memory Map — no UUIDs, no per-UUID counts. Bucketing of active_agents_now (`active_agents_bucket` field) defends against fingerprinting heavy users (per CONTEXT Claude's Discretion). Aggregate counts < k=5 are not separately defended at the active-tracker boundary because the active window is ephemeral (≤10 min) and re-derivation from event log is already gated by the housekeeper's k-floor; the worst-case adversary observation is "between 1 and 4 people are currently using FSB" which carries low re-identification value. |
| T-274-03 | Denial of Service | public-stats GET endpoint (no rate limit) | mitigate | 30-second in-process memo means repeated GETs cost ~O(1) memory lookups, never SQLite reads. `Cache-Control: max-age=60` lets edge caches absorb load. If load nevertheless rises, the existing Fly.io proxy + extension rate-limit budget would absorb spikes. No new rate-limiter mounted here — adding one would discourage public consumption by third-party metrics aggregators (aligned with STATS-07 "endpoint exists but is internal-use-only"). |
| T-274-04 | Denial of Service | active-tracker memory growth | mitigate | Lazy eviction on every read drops entries older than 10 minutes. Worst case: process running with no GET traffic accumulates UUIDs faster than they expire. Counter-argument: every POST to /api/telemetry/events comes through the per-UUID daily-budget gate (1000/day), so the upper bound on distinct UUIDs in the Map per 24h is `(total_active_uuids)`; for v0.9.69 install-base this is bounded in the low thousands. If install-base grows past ~100k DAU, switch to a TTL-based LRU. Not in scope for this milestone. |
| T-274-05 | Tampering | If-None-Match header (untrusted) | accept | The header is compared as a string against our in-process memo's stored ETag. A malicious header value either matches (→ 304) or doesn't (→ 200). Neither path runs SQL nor mutates state. The ETag itself is a SHA-256 prefix of the response body, so forging it would require predicting the cache key — and even if forged, the worst outcome is a 304 with empty body (no data exfiltration). |
| T-274-06 | Spoofing | Counterfeit `recordSeen` calls | accept | recordSeen is module-private (only called from telemetry.js after validated insert). No external entry point.
