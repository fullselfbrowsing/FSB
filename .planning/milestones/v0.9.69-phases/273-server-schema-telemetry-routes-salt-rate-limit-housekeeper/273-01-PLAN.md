Land the showcase server's anonymous telemetry ingest pipeline with three release-gating safeguards:

1. **BLOCKER #1** — `app.set('trust proxy', 1)` placed IMMEDIATELY after `const app = express()`, BEFORE any middleware or route mount, so `req.ip` returns the real client IP (not the Fly.io single-edge proxy IP).

3. **Privacy invariant** — plaintext IP read once from `req.ip`, immediately hashed with a daily-rotated salt stored in `telemetry_daily_salt`, plaintext discarded the same statement; a CI grep gate (`tests/server-no-ip-leak.test.js`) fails the build if any morgan/winston/express-logger middleware is mounted in `showcase/server/src/**/*.js` without a redact filter.

Without it, every install collapses into Fly's proxy IP (BLOCKER #1) and the rate limiter mis-buckets dual-stack hosts (BLOCKER #2). Downstream Phase 274 reads the aggregates this pipeline produces.

Output:
- 4 new SQLite tables (telemetry_events, telemetry_rollups_daily, telemetry_global_aggregates, telemetry_daily_salt) with proper indexes and PRAGMAs.
- 3 unauthenticated POST endpoints (`/api/telemetry/events`, `/api/telemetry/optout`, `/api/telemetry/forget`) layered with rate limit + body cap + batch cap + strict allowlist + timestamp tolerance + per-UUID budget + Sec-GPC honoring.

**1e. `showcase/server/src/utils/telemetry-salt.js` (INGEST-04):**
Create new file. CJS module. Exports `getOrMintTodaySalt(db) -> {salt_hex, day_utc}`.

```
Algorithm:
  1. const day_utc = new Date().toISOString().slice(0,10)   // 'YYYY-MM-DD' in UTC
  2. const row = queries.selectTodaySalt.get(day_utc)        // SELECT salt_hex, minted_at
  3. if (row) return { salt_hex: row.salt_hex, day_utc }
  4. ELSE: BEGIN IMMEDIATE TRANSACTION (better-sqlite3 transaction() helper):
       a. Re-check inside tx (concurrent boot race)
       b. INSERT today: salt_hex = crypto.randomBytes(32).toString('hex'), minted_at = Date.now()
       c. const yesterday = previous UTC day string
       d. DELETE FROM telemetry_daily_salt WHERE day_utc < yesterday    -- retains today + yesterday only
       e. COMMIT
  5. Return { salt_hex, day_utc }
```

Take a `db` (better-sqlite3 instance) parameter directly. Do NOT import queries.js here — this module is a primitive used by both queries.js and tests. Build the 3 prepared statements (`selectTodaySalt`, `insertTodaySalt`, `deleteOldSalts`) lazily on first call and cache them on a module-level `WeakMap<db, statements>` so the same db handle reuses the same prepared statements. (Per STACK §4.) The yesterday computation: `new Date(Date.now() - 86400000).toISOString().slice(0, 10)`.

Why lazy module-level WeakMap, not queries.js: the housekeeper, the rate-limit middleware, and the routes all want salt, but the salt module is a SHARED PRIMITIVE — coupling it to the Queries class would force a circular import (Queries -> salt -> Queries). Direct `db.prepare(...)` keeps the dependency graph a DAG.

**1f. `showcase/server/src/utils/telemetry-hash.js` (INGEST-03, D-09):**
Create new file. CJS module. Exports `hashIp(plaintextIp, db) -> hex_string`.

```
Algorithm:
  1. const { salt_hex } = getOrMintTodaySalt(db)
  2. const key = Buffer.from(salt_hex, 'hex')
  3. const h = crypto.createHmac('sha256', key).update(plaintextIp, 'utf8').digest('hex')

```

CRITICAL: do NOT assign `plaintextIp` to any module-level variable. Do NOT log it. Do NOT pass it anywhere except into the `update()` call. The `plaintextIp` parameter goes out of scope as soon as `hashIp` returns. The function signature reads `req.ip` as an inline argument from the call site — never assigned to a variable that escapes.

Also export an `isValidUuidV4(str) -> bool` helper using `/^[0-9a-f]{8}-[0-9a-f]{4}-4[0-9a-f]{3}-[89ab][0-9a-f]{3}-[0-9a-f]{12}$/i` for use by the routes module.
