<objective>
Author `scripts/verify-classification-gate.mjs` — a fail-closed build-time/CI gate that, for each origin a descriptor corpus would emit, calls `service-denylist.js classify(origin)` and FAILS the build when an origin trips a sensitivity heuristic (finance/payment/health/social/media/adult/government keyword) but is not explicitly classified denied or sensitive. Genuinely-benign origins default to safe. Mirror the established `verify-recipe-path-guard.mjs` CI-gate shape (Node-builtins-only, `failures[]` accumulator, `process.exit(1)` on fail). Chain it into `validate:extension`. Add a paired `tests/classification-gate.test.js` and a deliberately-unclassified sensitive fixture proving fail-closed.

Purpose: "a gap in the JSON array is indistinguishable from an allow decision" must become a CI failure, not a silent allow. This gate is authored here (Phase 35) and CONSUMED by the Phase-36 importer, which calls `classifyGate()` before emitting any descriptor JSON — so it must export a reusable function AND run as a CLI.

Output: `scripts/verify-classification-gate.mjs` (with a `classifyGate` export), the fixture, the CI test, and the `validate:extension` chain extension.
</objective>
