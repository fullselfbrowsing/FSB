## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Self-inflicted grafana overwrite from a zsh word-splitting bug in the vendoring loop**
- **Found during:** Task 1 (the vendor copy loop)
- **Issue:** The Bash tool runs under zsh, which (unlike bash) does NOT word-split an unquoted multi-word variable. My preserve skip-list `is_in "$app" $PRESERVE` therefore passed the entire 14-word list as a single argument and never matched, so the real grafana slice overwrote the hand-authored grafana.com slice (the single-word `$NEVER`=sqlpad matched fine, masking the bug).
- **Fix:** Detected immediately via the post-run check (13 sdk-stubs instead of 14; `git status` showed grafana M/D/??). Surgically restored the hand-authored grafana from HEAD (`rm -rf .../grafana && git checkout HEAD -- .../grafana`). The 13 hand-only dirs were never in the upstream loop, so they were unaffected (git confirmed no changes). grafana is the ONLY upstream app in the preserve set, so the single restore made the vendored set correct.
- **Files modified:** vendor/opentabs-snapshot/plugins/grafana/ (restored to hand-authored)
- **Verification:** grafana urlPatterns back to `*://grafana.com/*`, sdk-stub.ts present, 14 sdk-stubs total, git clean vs HEAD; the new guard test's Test 3 now asserts grafana.com specifically (catches this exact regression).
- **Committed in:** f00d560f (Task 1 -- grafana was restored before staging, so the commit contains the correct augment-preserved state)

**2. [Rule 3 - Blocking] provenance-scaffold.test.js conflicted with the required _provenance.json population**
