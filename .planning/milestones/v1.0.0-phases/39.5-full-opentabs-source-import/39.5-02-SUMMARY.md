## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] tsx subprocess for the real-plugin carve proof**
- **Found during:** Task 3 (carve proof test)
- **Issue:** The plan directs the carve proof to use `extractDescriptors('target')`, which imports the real `target/src/index.ts` (whose sibling `.js` specifiers resolve to `.ts`). Under plain `node` (Node 24) this fails with `Cannot find module ...target-api.js`; it requires the tsx loader, exactly as the importer documents (`node --import tsx`). `tests/import-forbidden-prescan.test.js` runs under plain `node` in the npm chain, and the plan's own verify command is plain `node tests/import-forbidden-prescan.test.js`.
- **Fix:** The test runs the `extractDescriptors('target')` call in a `node --import tsx -e ...` subprocess and parses the emitted params from stdout, keeping the test file itself runnable under plain `node` (so the plan's verify command and the npm chain are unchanged). The pure-helper checks (preScanForbidden/assertCleanParams) run in-process as before.
- **Files modified:** tests/import-forbidden-prescan.test.js
- **Verification:** `node tests/import-forbidden-prescan.test.js` exits 0 (24 checks) under plain node; the subprocess proves the real descriptor emits `promo_code` and no `code`.
- **Committed in:** `b99be2c7` (Task 3 commit)

**2. [Rule 1 - Bug] Duplicate `importerUrl` declaration in the extended test**
- **Found during:** Task 3 (first run of the extended test)
- **Issue:** The carve block redeclared `const importerUrl`, which already exists at the top of `main()` -> `SyntaxError: Identifier 'importerUrl' has already been declared`.
- **Fix:** Removed the redeclaration; reused the existing `importerUrl` (same importer module URL).
- **Files modified:** tests/import-forbidden-prescan.test.js
- **Verification:** test exits 0 (24 checks).
- **Committed in:** `b99be2c7` (Task 3 commit, pre-commit)
