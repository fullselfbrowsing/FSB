- **Added the build-failing backstop:** `COMMERCE_SENSITIVE_SERVICES` (32 emitted hosts) + exported `checkCommerceSensitiveClassified(classifyFn)` in verify-catalog-crosscheck.mjs -- the inverse of READ_ONLY_SAFE_SERVICES/checkReadOnlySafeOrigins. Wired into runCli so `npm run validate:extension` FAILS on an omitted/downgraded commerce brand. Corpus-independent (pure roster + classify()), so it catches a read-only commerce brand neither classifyGate (no payment token) nor the payment-op guard (no payment verb) can catch.
- **Proved it:** tests/full-corpus-screen.test.js drives the REAL classifyGate over every enumerated real-app origin (127 apps -> 0 failures, corpus classify-complete), the REAL backstop over the full non-empty roster (0 failures), an explicit non-vacuity check (roster size 32 >= 28) and a gate-bites check (a downgraded homedepot.com yields a failure). validate:extension exit 0, 228 committed descriptors unchanged.
