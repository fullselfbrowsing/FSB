<objective>
Make every searchable descriptor invocable (CGEN-03): add the SINGLE fallback branch in
`capability-catalog.js resolve()` so a slug that is searchable (in `FsbRecipeIndex.descriptors`)
but has NO REGISTRY handler and NO recipe resolves to a non-null SEAM tier — `{tier:'T3'}` (DOM)
by default, `{tier:'T2'}` (learn-pending) when the descriptor's `backing === 'learn'` — instead
of returning null and re-introducing `RECIPE_NOT_FOUND` for a slug search can surface. Prove the
invariant with a harness that asserts every corpus slug resolves to a real seam tier and an
out-of-corpus slug resolves to null, and extend the router test to prove an actual invoke yields
`RECIPE_DOM_FALLBACK_PENDING`/`RECIPE_LEARN_PENDING`, never `RECIPE_NOT_FOUND`.

Purpose: this is THE quality gate of Phase 36 and the only load-bearing runtime edit. Without it,
the moment descriptors land they are searchable-but-uninvocable dead entries (the headline risk).
The router already maps T3/T2 to typed reasons — zero router changes; the fallback only needs to
emit the EXACT tier strings the router's switch matches.

Output: the resolve() branch + the `_getDescriptor`/`_recipeIndex` accessors;
`tests/no-dead-entry.test.js`; an extended `tests/capability-router.test.js`.
</objective>
