Plan 12-02 (Wave 2) wires the addMessage chokepoint to write through to the new persistence store via the Plan 12-00 sidecar debouncer.

Per CONTEXT D-03: write-through every `addMessage` call, debounced 200ms per conversationId, clear-and-replace timer reset. Per CONTEXT D-04: hard LRU cap of 50 conversations. Per CONTEXT D-10: every message that lands in the DOM via addMessage MUST also land in `fsbConversationMessages`. Per RESEARCH Section 5: addMessage is the canonical chokepoint; addCompletionMessage + addActionMessage are sibling chokepoints that must also hook.

Architecture:
1. Module-scope `_messageLogDebouncer` initialized at boot.
2. New private `_persistMessage(role, content, kind)` helper invoked from addMessage + addCompletionMessage + addActionMessage AFTER existing DOM render.
3. `_persistMessage` schedules a per-convId flush via `_messageLogDebouncer.schedule(convId, flushFn)` where `flushFn` reads the envelope from chrome.storage.local, appends the in-memory buffer via `FSBSidepanelMessageLog.appendMessage`, persists.
4. In-memory buffer keyed by convId collects messages between flushes (so a burst of 5 messages in 100ms results in 1 storage write at 200ms after the last one, batched).
5. `window.addEventListener('beforeunload', ...)` calls `_messageLogDebouncer.flushAll()` (best-effort; the page is closing).

Plan 12-02 ALSO routes `dropConversationMessages` from chrome.tabs.onRemoved (the Phase 11 onRemoved listener already calls `dropTabConversation`; Plan 12-02 extends it to call `dropConversationMessages` + `_messageLogDebouncer.cancel(convId)` to defeat the EC-05 resurrection-after-drop edge case).

Pitfall 3 defense from Plan 12-01: `renderPersistedMessage` continues to be the hydrate render path so re-hydrating does NOT trigger write-through (only `addMessage` does, and hydrate doesn't call addMessage).

## `_persistMessage` helper shape

```js
/**
 * Plan 12-02 FINT-23 write-through helper.
 *
 * Called from addMessage + addCompletionMessage + addActionMessage AFTER
 * the existing DOM render. Schedules a 200ms-debounced flush per convId
 * via the module-scope _messageLogDebouncer.
 *
 * Guards:
 *  - return early if conversationId is null/empty (lazy-mint window;
 *    early boot messages before convId is minted skip persistence).
 *  - return early if content is empty.
 *  - return early if FSBSidepanelMessageLog is undefined (script-tag
 *    load order failure).
 *
 * Storage failures swallow silently -- DOM render must never block on
 * persistence per CONTEXT D-03.
 */
function _persistMessage(role, content, kind) {
  if (typeof FSBSidepanelMessageLog === 'undefined') return;
  if (!conversationId || typeof conversationId !== 'string') return;
  if (typeof content !== 'string' || content.length === 0) return;
  if (!_messageLogDebouncer) return;
  // Validate role -- treat 'system' / 'error' / 'action' DOM-type literals
  // as 'assistant' role per the schema in CONTEXT D-02.
  var resolvedRole = (role === 'user') ? 'user' : 'assistant';
  var resolvedKind = kind || 'text';
  // Append to in-memory buffer immediately (read consistency within debounce window).
  var convId = conversationId;
  var buffer = _messageLogPendingBuffer.get(convId);
  if (!buffer) {
    buffer = [];
    _messageLogPendingBuffer.set(convId, buffer);
  }
  buffer.push({
    role: resolvedRole,
    content: content,
    timestamp: Date.now(),
    kind: resolvedKind
  });
  // Schedule debounced flush. Clear-and-replace (D-03).
  _messageLogDebouncer.schedule(convId, function () {
    return _flushMessageLog(convId);
  });
}

async function _flushMessageLog(convId) {
  var buffer = _messageLogPendingBuffer.get(convId);
  if (!buffer || buffer.length === 0) return;
  // Snapshot + clear in place so concurrent appends during await go to next batch.
  var snapshot = buffer.slice();
  buffer.length = 0;
  try {
    var bag = await chrome.storage.local.get(FSBSidepanelMessageLog.STORAGE_KEY);
    var envelope = bag[FSBSidepanelMessageLog.STORAGE_KEY];
    if (!FSBSidepanelMessageLog.isValidEnvelope(envelope)) {
      envelope = FSBSidepanelMessageLog.emptyEnvelope();
    }
    for (var i = 0; i < snapshot.length; i++) {
      FSBSidepanelMessageLog.appendMessage(envelope, convId, snapshot[i]);
    }
    var payload = {};
    payload[FSBSidepanelMessageLog.STORAGE_KEY] = envelope;
    await chrome.storage.local.set(payload);
  } catch (_e) {
    // Best-effort: storage failure resurrects the buffer so next flush retries.
    var current = _messageLogPendingBuffer.get(convId);
    if (current) {
      _messageLogPendingBuffer.set(convId, snapshot.concat(current));
    } else {
      _messageLogPendingBuffer.set(convId, snapshot);
    }
  }
}
```
