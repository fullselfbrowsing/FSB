## Milestone Definition Of Done

v0.12.0 replaces FSB's in-house Phantom Stream implementation with the published `@full-self-browsing/phantom-stream@0.1.0` package while preserving the dashboard live preview, stream recovery, side channels, relay behavior, and remote-control path.

The milestone explicitly excludes MCP schema changes, dashboard redesign, task/status WebSocket replacement, pixel/video streaming, and publishing PhantomStream itself.

Live browser UAT remains user-gated:

- dashboard live preview starts and stays nonblank;
- mutation fidelity under real interaction;
- scroll, overlay, progress/client badge, and dialog side channels;
- remote click/type/key/scroll through CDP;
- navigation/retarget and stale-frame rejection;
- dashboard reconnect and extension wake/reinject recovery;
- restricted/no-tab states;
- large-page behavior;
- security masking in Chrome;
- external debugger contention.

These are explicitly listed in `.planning/phases/25-parity-removal-docs-browser-uat/25-HUMAN-UAT.md`.

The milestone is ready for closeout with recorded tech debt. It should not be represented as fully live-browser validated until the `human_needed` scenarios are executed and updated with observed outcomes.
