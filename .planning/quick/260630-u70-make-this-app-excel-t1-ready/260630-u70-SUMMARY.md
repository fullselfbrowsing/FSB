# Make This App Excel T1-Ready

## Result

Excel Online now has explicit T1 accounting across all 29 `excel.*` descriptors:

- 10 safe read rows resolve as `t1-ready` through a bundled T1a handler.
- 19 side-effecting rows resolve as guarded fail-closed and return dual-field `RECIPE_DOM_FALLBACK_PENDING` without dispatching execution primitives.
- `excel.reauthenticate` is classified as `write`, because it clears auth state and reloads the tab.
- The canonical handler and unpacked extension copy match byte-for-byte.

## Implementation Notes

- Added `catalog/handlers/excel.js` and `extension/catalog/handlers/excel.js`.
- Excel reads use only bounded page-read auth context plus bounded Microsoft Graph GET specs.
- Excel auth context is read inside the origin-pinned page context, from captured/mirrored Graph token state or Graph MSAL localStorage, and returns a typed fallback when token or workbook context is missing.
- SharePoint workbook URLs are handled by resolving the sharing URL through Graph before workbook reads.
- No direct handler `fetch`, `chrome.*`, page storage, cookies, or token logging path was added.
- Runtime/import, head manifest, readiness/search overrides, reports, path guard, port contract, origin classifier, generated recipe index, and write-activation evidence were updated.
