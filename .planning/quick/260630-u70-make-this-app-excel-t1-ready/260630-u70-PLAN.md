# Make Excel T1-ready where safe

## Objective

Promote the existing Excel Online catalog stem from discovery-pending to explicit T1 accounting: safe workbook/user reads become reviewed T1a handlers, while workbook mutations, destructive actions, formula-evaluation temp-cell writes, and reauthentication stay guarded fail-closed.

## Source Coverage Audit

- `.planning/STATE.md`: preserve the two-tool capability surface, MV3 data/code wall, honest T1 readiness accounting, and fail-closed write posture.
- MS Word and PowerPoint quick plans: follow the Microsoft Graph T1 pattern with app handler code under `catalog/handlers/`, byte-matching extension copy, runtime/head manifest wiring, readiness/search/report wiring, guarded write evidence, and focused tests.
- Excel descriptors: cover all 29 `excel.*` rows. Promote only the 10 pure read rows; guard the remaining 19 rows.
- Dirty worktree: many unrelated T1 migrations are already modified or untracked. Do not revert, reset, normalize, or overwrite unrelated dirty changes. If a shared file is already dirty, make the smallest contextual edit and record the dirty-state caveat in the summary instead of trying to force an atomic commit.

## Safe Read Slugs

Register exactly these read descriptors as T1a:

`excel.get_current_user`, `excel.get_range`, `excel.get_table_columns`, `excel.get_table_rows`, `excel.get_used_range`, `excel.get_workbook_info`, `excel.list_charts`, `excel.list_named_items`, `excel.list_tables`, `excel.list_worksheets`.

Guard exactly these rows as fail-closed:

`excel.add_named_item`, `excel.add_table_column`, `excel.add_table_row`, `excel.add_worksheet`, `excel.calculate_workbook`, `excel.clear_range`, `excel.create_chart`, `excel.create_table`, `excel.delete_chart`, `excel.delete_range`, `excel.delete_table`, `excel.delete_table_row`, `excel.delete_worksheet`, `excel.evaluate_formula`, `excel.insert_range`, `excel.reauthenticate`, `excel.sort_range`, `excel.update_range`, `excel.update_worksheet`.

`excel.evaluate_formula` must stay guarded because the upstream implementation writes a formula into a temp cell, reads it, then clears it. `excel.reauthenticate` must be corrected from read-like treatment to side-effecting guarded behavior because it clears MSAL/local token cache and reloads the page.
