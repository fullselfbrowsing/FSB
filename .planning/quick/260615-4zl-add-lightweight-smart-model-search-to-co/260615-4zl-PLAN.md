# Quick Task 260615-4zl: Smart Model Search In Control Panel

## Objective

Add a lightweight, reliable search box for the control panel model selector so large provider model lists can be narrowed quickly.

## Scope

- Add a model search input near the existing `#modelName` select.
- Filter discovered and fallback model lists in memory.
- Match case-insensitively across model id, display name, name, description, and provider.
- Require every query token to match somewhere in the model text so queries like `gpt free` work predictably.
- Preserve the existing model discovery fetch/cache flow; no network behavior changes.

## Acceptance

- Searching `free` shows models whose id/name/description contains `free`, including OpenRouter-style ids where `free` appears at the end.
- Searching `GPT` matches `gpt-*` / `GPT-*` models case-insensitively.
- Empty search restores the full model list.
- No matching query renders a disabled no-results option.
- `node tests/model-discovery-ui.test.js` passes.
