<objective>
Fix UAT-1 CSP violation #2: the offscreen lattice-host bundle currently lists `node:*` modules as esbuild externals, which ESM-target builds preserve as live `import 'node:fs/promises'` (etc.) statements in the output. Chrome MV3 enforces `script-src 'self'` on offscreen documents and rejects these specifiers at parse time, preventing the lattice-host offscreen document from loading.

Purpose: Replace the external-array approach with an inline esbuild plugin that resolves any `node:*` specifier to a local stub module at build time. Stubs export no-op async functions for the artifact-storage surface (mkdir/readFile/readdir/rm/stat/writeFile/join/fileURLToPath/default). The artifact-storage code paths inside Lattice are dead at runtime in our offscreen context, but their top-level imports must still be satisfied — locally — so the bundle parses under MV3 CSP.

Output: An updated `esbuild.config.js` (offscreen-lattice-host entry only), and a regenerated `extension/dist/offscreen/lattice-host.js` + sourcemap that contains zero surviving `node:` import specifiers. Source files in `extension/offscreen/` and `./lattice/` are not touched.
</objective>

<rationale>
This is the second Phase 5 deferred-task bug surfaced during UAT-1.

  - First (already fixed in quick task 260531-63l): offscreen HTML src path pointed at
    unbundled source.
  - Second (this task): esbuild config marked `node:*` as external. ESM externals are
    preserved as live `import` statements in browser-target output and trigger CSP
    `script-src 'self'` violations.

Root cause class: the Phase 5 author assumed tree-shaking would remove the unreachable
artifact-storage code paths in Lattice. ESM top-level imports cannot be tree-shaken
merely because they're marked external — esbuild preserves the import specifier
verbatim. Browser ESM bundles require build-time stubbing of every `node:*` specifier,
even if the importing module's exports are unreferenced.

INV-06 constraint: do NOT modify Lattice. The current_lattice_sha
`e95067bfa87ed1b75838fc3b3ef217a3b01acbd3` must stay pinned. The fix is entirely
inside the FSB esbuild config.
</rationale>

<current_problem_snippet>
esbuild.config.js lines 89-95 (the lines being replaced):

```javascript
    // [Rule 3 - Blocker] Lattice dist/index.js top-level imports node:fs/promises,
    // node:path, node:url for its artifact-storage submodule. Our offscreen import
    // surface (checkpoint/signer/survivability) does NOT touch those code paths.
    // Marking node:* as external lets the bundle compile; tree-shaking removes the
    // unreachable artifact code at minify time. The empty external stubs will never
```

The above comment was wrong: `external` does NOT cause tree-shaking; it preserves the
import specifier as-is. The new approach (plugin-based stubbing) replaces both the
comment block and the external array.
