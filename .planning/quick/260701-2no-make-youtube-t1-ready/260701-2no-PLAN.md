# Quick Task 260701-2no: Make YouTube T1-ready

## Goal

Make YouTube's T1 terminal state explicit under the current policy boundary.

`youtube.com` and `www.youtube.com` are denied origins in `extension/config/service-denylist.json`. Because readiness generation classifies denied origins as `blocked` before handler or guarded overrides can make them executable, this task must not add a YouTube runtime handler, recipe, or search readiness override. The correct T1 outcome is checked blocked-policy terminal coverage: every YouTube descriptor remains non-invocable until a product/legal denylist change exists.

## Scope

- Confirm all 18 `youtube.*` descriptors report `readiness: blocked`, `originClass: denied`, `routeFeasibility: blocked`, `nextAction: keep blocked`, and no handler/recipe proof.
- Confirm the terminal-state report surfaces YouTube as `blocked-policy` with `executionEnabled: false`.
- Confirm the write/destructive YouTube ledger rows are `blocked-policy` with `activationAllowed: false`.
