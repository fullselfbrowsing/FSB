# Quick 260630-u6d Plan: YNAB T1 Readiness

## Objective

Make the existing `ynab.*` catalog descriptors T1-ready for `https://app.ynab.com`: safe budget/account/category/payee/month/transaction reads become reviewed T1a handlers, and all budget mutations stay visible but guarded fail-closed until live mutation-body UAT exists.

## Scope

T1-ready reads: `ynab.get_account`, `ynab.get_current_user`, `ynab.get_month`, `ynab.get_plan`, `ynab.get_transaction`, `ynab.list_accounts`, `ynab.list_categories`, `ynab.list_months`, `ynab.list_payees`, `ynab.list_scheduled_transactions`, `ynab.list_transactions`.

Guarded fail-closed rows: `ynab.create_category`, `ynab.create_category_group`, `ynab.create_transaction`, `ynab.delete_category`, `ynab.delete_category_group`, `ynab.delete_transaction`, `ynab.move_category_budget`, `ynab.snooze_category_goal`, `ynab.update_category`, `ynab.update_category_budget`, `ynab.update_transaction`.

`ynab.snooze_category_goal` is currently read-classified in the descriptor set but calls `syncWrite` upstream. Correct it to `sideEffectClass: "write"` before registering it as guarded.
