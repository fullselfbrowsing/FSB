# Implement Datadog T1 Readiness

## Objective

Promote Datadog's reviewed same-origin GET read surface to bundled T1a handler execution while keeping Datadog writes, destructive actions, and POST-shaped read/search operations non-executable until live body/write evidence exists.

## Scope

- Datadog-owned handler code in `catalog/handlers/datadog.js` and packaged mirror `extension/catalog/handlers/datadog.js`.
- Datadog-only descriptor/readiness/search/test wiring needed for the handler to resolve as T1a.
- GSD quick artifacts and `STATE.md` quick-task tracking.
- Existing parallel-agent changes must be preserved; do not revert or reformat shared files.

## Promoted Slugs

Promote only the GET-backed Datadog reads from the vendored snapshot:

`datadog.get_current_user`, `datadog.get_dashboard`, `datadog.get_downtime`, `datadog.get_host_info`, `datadog.get_host_totals`, `datadog.get_incident`, `datadog.get_metric_metadata`, `datadog.get_monitor`, `datadog.get_monitor_groups`, `datadog.get_notebook`, `datadog.get_org_config`, `datadog.get_permissions`, `datadog.get_service_definition`, `datadog.get_service_dependencies`, `datadog.get_slo`, `datadog.get_slo_history`, `datadog.get_synthetics_results`, `datadog.get_synthetics_test`, `datadog.get_trace`, `datadog.get_usage_summary`, `datadog.get_user`, `datadog.list_api_keys`, `datadog.list_dashboards`, `datadog.list_downtimes`, `datadog.list_host_tags`, `datadog.list_hosts`, `datadog.list_incidents`, `datadog.list_metric_tags`, `datadog.list_metrics`, `datadog.list_monitor_downtimes`, `datadog.list_monitor_tags`, `datadog.list_monitors`, `datadog.list_notebooks`, `datadog.list_services`, `datadog.list_slo_corrections`, `datadog.list_slos`, `datadog.list_synthetics_tests`, `datadog.list_teams`, `datadog.list_users`, `datadog.query_metrics`, `datadog.search_dashboards`, `datadog.search_dashboards_advanced`, `datadog.search_monitors`, `datadog.search_notebooks`, `datadog.search_services`, `datadog.search_slos`.
