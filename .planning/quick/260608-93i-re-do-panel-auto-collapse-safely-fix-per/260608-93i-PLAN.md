Two related tab-scoping fixes in the FSB extension:

**Issue A: Re-do panel auto-collapse safely (QT-93i-01).** The prior attempt (commit 779bbae2) was reverted in b7cb5283 because `await chrome.sidePanel.setOptions(...)` BEFORE `chrome.sidePanel.open(...)` inside `chrome.action.onClicked` consumed the user-gesture token, causing `open()` to reject and fall through to the `chrome.windows.create({ type: 'popup' })` branch ("opens as a full-screen / new page" symptom).

This redo lands the SAME outcome (per-tab auto-collapse via `chrome.tabs.onActivated` + `chrome.sidePanel.setOptions`) but using the **fire-both-Promises-synchronously, await later** pattern inside `chrome.action.onClicked` so the gesture token is preserved through the `open()` call. The `chrome.tabs.onActivated` listener can still `await` because that event is NOT in user-gesture context.

**Issue B: Per-tab isRunning state (QT-93i-02).** Module-scope `let isRunning = false` and `let currentSessionId = null` in sidepanel.js make the send-button gating GLOBAL across tabs. Dispatching on tab A then swapping to tab B leaves sendBtn DISABLED on B even though B has no in-flight work. Replace with a per-tab `Map<tabId, { isRunning, sessionId }>` keyed by tabId, accessed via `getCurrentTabRunningState()` from `updateSendButtonState`. On tab-swap (existing `chrome.tabs.onActivated` handler at sidepanel.js:786 that already calls `swapToTabConversation`), re-resolve the UI from the new active tab's per-tab state. The `automationComplete` / `iteration_complete` handlers must route the running-state transition (setIdleState) by the **originating tab** (via `request.tabId` or session->tab lookup), NOT the currently-displayed tab.

Purpose: Per-tab panel visibility matches "has work running here" (Issue A); per-tab send-button gating matches "is THIS tab working" (Issue B). Honors CONTEXT D-01 (all-working-tabs-enabled) + D-02 (force-open with welcome state) from the prior 7bi context -- those decisions STILL APPLY.

## Pitfalls -- read before editing

1. **The 779bbae2 bug.** `await chrome.sidePanel.setOptions(...)` BEFORE `chrome.sidePanel.open(...)` inside `chrome.action.onClicked` consumes the gesture token. `open()` then rejects (`User gesture is required`) and the catch branch falls through to `chrome.windows.create({ type: 'popup' })`. Symptom: "panel opens as a full-screen / new page".

   **Correct pattern** (use this verbatim):
   ```js
   chrome.action.onClicked.addListener(async (tab) => {
     armMcpBridge('action.onClicked');
     automationLogger.logInit('sidepanel', 'opening', { windowId: tab.windowId });

     // Fire BOTH calls synchronously inside the gesture window so the
     // open call's user-gesture context is preserved. Both Promises are
     // created BEFORE any await.
     let setOptionsPromise = null;
     let openPromise = null;
     try {
       if (tab && typeof tab.id === 'number'
           && typeof chrome.sidePanel !== 'undefined'
           && typeof chrome.sidePanel.setOptions === 'function') {
         setOptionsPromise = chrome.sidePanel.setOptions({
           tabId: tab.id,
           enabled: true,
           path: 'ui/sidepanel.html'
         });
       }
     } catch (_e) { /* sync throw -- log later */ }
     try {
       openPromise = chrome.sidePanel.open({ windowId: tab.windowId });
     } catch (_e) { /* sync throw -- log later */ }

     // Await OPEN first (gesture-critical, fail mode is popup fallback).
     try {
       if (openPromise) await openPromise;
       automationLogger.logInit('sidepanel', 'ready', { windowId: tab.windowId });
     } catch (error) {
       automationLogger.logInit('sidepanel', 'failed', {
         error: error && error.message, fallback: 'popup'
       });

         url: chrome.runtime.getURL('ui/popup.html'),
         type: 'popup', width: 400, height: 600
       });

     try {
       if (setOptionsPromise) await setOptionsPromise;
     } catch (setErr) {
       console.warn('[FSB] QT-93i-01 force-open setOptions failed', {
         tabId: tab && tab.id, error: setErr && setErr.message
   ```

```js
  var startIdx = src.indexOf(anchor);
  if (startIdx === -1) return null;
  var braceIdx = src.indexOf('{', startIdx);
  if (braceIdx === -1) return null;
  var depth = 1;

  while (i < src.length && depth > 0) {
```
