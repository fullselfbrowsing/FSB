# Quick Task 260630-ql3: Make ChatGPT T1-ready

## Goal

Promote ChatGPT's source-reviewable same-origin read surface from DOM/discovery tail to verified T1a handler coverage.

The safe surface is limited to read descriptors whose vendored OpenTabs tools call `https://chatgpt.com/backend-api` with GET requests. Do not activate ChatGPT write or destructive descriptors unless this same task records live mutation-body evidence and the guarded write gates are updated; this plan does not request that activation.

Current working-tree note: `catalog/handlers/chatgpt.js` and `extension/catalog/handlers/chatgpt.js` may already exist. Treat them as a starting point to review, harden, and wire; do not create a second handler surface.

## Source Audit

- GOAL: "Make this app ChatGPT T1-ready" is covered by handler-backed T1a coverage for the safe ChatGPT read subset and refreshed readiness/search surfaces.
- STATE invariants: keep the two-tool MCP surface, preserve MV3 Wall 1 and Wall 2, keep OpenTabs runtime/plugin code out of the extension, and keep writes/destructive actions non-executable without live mutation evidence.
- Vendor source: `vendor/opentabs-snapshot/plugins/chatgpt/src/chatgpt-api.ts` defines `API_BASE = "https://chatgpt.com/backend-api"` and `SESSION_URL = "https://chatgpt.com/api/auth/session"`. The promoted read tools use GETs; archive/delete/rename/star/unstar/update use PATCH or POST and must stay unregistered.
- Required gates from prompt: handler registration, extension copy/import, readiness/port-contract origin maps, head-handler cap, handler tests, and upgrade test entries are all included below.

- `chatgpt.discover_gpts`
- `chatgpt.get_account_info`
- `chatgpt.get_beta_features`
- `chatgpt.get_conversation`

- `chatgpt.list_models`

## Excluded Rows

Do not register these rows in `FsbHandlerChatgpt`: `chatgpt.archive_conversation`, `chatgpt.delete_conversation`, `chatgpt.rename_conversation`, `chatgpt.star_conversation`, `chatgpt.unarchive_conversation`, `chatgpt.unstar_conversation`, and `chatgpt.update_custom_instructions`.

### 1. Harden the ChatGPT same-origin read handler

Files:
- `catalog/handlers/chatgpt.js`
- `extension/catalog/handlers/chatgpt.js`
- `tests/capability-head-handlers.test.js`

Action:
- Implement or adjust the dual IIFE/CommonJS handler module exporting `FsbHandlerChatgpt`.
- Use `ORIGIN = "https://chatgpt.com"`, `SERVICE = "chatgpt.com"`, `API_BASE = ORIGIN + "/backend-api"`, and `SESSION_URL = ORIGIN + "/api/auth/session"`.
- Register only the 13 T1-ready slugs above as `tier: "T1a"`, `origin: ORIGIN`, `sideEffectClass: "read"`, with closed params copied from the existing descriptor JSON.
- Every handler call must use `ctx.executeBoundSpec`; never call direct `fetch`, `XMLHttpRequest`, `chrome.scripting`, `chrome.tabs`, `chrome.cookies`, `chrome.webRequest`, `document.cookie`, `localStorage`, or `sessionStorage`.
- Fetch the access token only through a first `executeBoundSpec` GET to `SESSION_URL` with `authStrategy: "same-origin-cookie"`, `origin: ORIGIN`, and `extract: "@"`. Do not log, persist, return, or cache the token.
- Use the token only in a second `executeBoundSpec` GET to `API_BASE + path`; never build a PATCH/POST/DELETE spec for ChatGPT in this task.
- Implement these endpoint mappings:
  - `discover_gpts`: GET `/gizmos/discovery?cursor={cursor || 0}&limit={limit || 10}&locale={locale || "en-US"}`
  - `get_account_info`: GET `/accounts/check/v4-2023-04-27`
  - `get_beta_features`: GET `/settings/beta_features`
  - `get_conversation`: GET `/conversation/{conversation_id}`
  - `get_current_user`: GET `/me`
  - `get_custom_instructions`: GET `/user_system_messages`
  - `get_gpt`: GET `/gizmos/{gpt_id}`
  - `get_memories`: GET `/memories`
  - `get_prompt_library`: GET `/prompt_library/`
  - `list_conversations`: GET `/conversations?offset={offset || 0}&limit={limit || 28}&order={order || "updated"}`
  - `list_models`: GET `/models?history_and_training_disabled=false`
  - `list_shared_conversations`: GET `/shared_conversations?offset={offset || 0}&limit={limit || 25}`
  - `search_conversations`: GET `/conversations/search?query={query}&limit={limit || 28}`
- Map response shapes using `vendor/opentabs-snapshot/plugins/chatgpt/src/tools/schemas.ts`: user, conversation list item, conversation detail active branch, model, memory, GPT, prompt, account info, beta features, custom instructions, and search result snippets.
- Return dual-field `RECIPE_DOM_FALLBACK_PENDING` with `fellBackToDom: true` on missing `ctx.executeBoundSpec`, missing access token, redirects, 401/403, HTTP errors, error envelopes, or missing required response fields.
- Copy `catalog/handlers/chatgpt.js` byte-for-byte to `extension/catalog/handlers/chatgpt.js`.
- Extend `tests/capability-head-handlers.test.js` with ChatGPT coverage: all 13 read slugs exported, all 7 excluded rows absent, session spec first, backend spec second, both specs pinned to `https://chatgpt.com`, backend URL starts with `https://chatgpt.com/backend-api/`, backend method is GET, no mutation methods appear, mapped outputs match fixtures, malformed session/backend shapes fall back, token is not logged/persisted, and extension copy parity includes `chatgpt.js`.
