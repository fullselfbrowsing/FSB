# Implement Gemini T1 readiness

## Summary

Gemini now has a conservative T1 handler surface:

- `gemini.get_current_user`, `gemini.list_models`, `gemini.list_conversations`, and `gemini.get_conversation` are same-origin `https://gemini.google.com` T1a read handlers using `executeBoundSpec`.
- `gemini.create_conversation` and `gemini.send_message` are T1a guarded fail-closed writes returning the dual-field `RECIPE_DOM_FALLBACK_PENDING` response without invoking execution primitives.
- Gemini is wired into background handler loading, handler catalog registration, readiness reporting/search status, sensitive origin classification, origin/path/port verification maps, and focused tests.
