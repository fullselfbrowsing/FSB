# Quick Task 260701-1kg: Make Walmart T1-ready

## Goal

Promote Walmart's safe public page-read subset from DOM/discovery tail to verified T1a handler coverage.

This task must not activate Walmart account, cart, order history, checkout, browser navigation, payment, or authenticated customer-profile flows. Walmart is a retail app with account/payment adjacency; only parameter-driven public page reads with source-reviewable `__NEXT_DATA__` contracts may become executable here.

## Source Audit

- GOAL: "Make this app Walmart T1-ready" is covered by adding handler-backed T1a coverage for Walmart's safe param-driven public read subset and refreshing readiness/search surfaces.
- STATE invariants: keep the two-tool capability surface, MV3 Wall 1, Wall 2 origin pin, sensitive-origin handling, and fail-closed write posture. No per-app MCP tool, OpenTabs runtime code, direct fetch, or generic credential bridge is introduced.
- Phase 44/51: Walmart currently has DOM-backed descriptors. This plan promotes only source-reviewable public reads; other Walmart rows stay non-executable with honest terminal-state accounting.
- Existing patterns: follow the current quick-task T1 head shape used by `target.js` and `costco.js`: dual IIFE/CommonJS handler, extension copy parity, `executeBoundSpec` only, app-specific origin proof, readiness overrides, descriptor backing alignment, and focused negative controls.

## T1-ready slugs

- `walmart.search_products`
- `walmart.get_product`
- `walmart.get_product_reviews`
- `walmart.get_store`

## Excluded rows

Do not register these rows in `FsbHandlerWalmart`: `walmart.get_cart`, `walmart.get_current_user`, `walmart.list_orders`, `walmart.navigate_to_checkout`, `walmart.navigate_to_product`, and `walmart.navigate_to_search`.

Rationale: current-user and order rows depend on authenticated customer state and cookie/auth-cache helpers; cart and checkout expose shopping/payment state; navigation rows call `window.location` and mutate page location rather than returning data. They need separate review or live UAT evidence before direct execution.
