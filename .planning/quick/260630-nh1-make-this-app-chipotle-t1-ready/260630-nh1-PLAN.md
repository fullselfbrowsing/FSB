# Quick Task 260630-nh1: Make Chipotle T1-ready

## Goal

Promote Chipotle's safe public restaurant/menu/status read surface from DOM/discovery tail to verified T1a handler coverage.

This task must not activate the authenticated Chipotle runtime that reads `cmg-vuex`, bearer JWTs, saved payment data, loyalty state, recent orders, favorites, or customer profile data. The vendored helper routes most Chipotle tools through `https://services.chipotle.com` plus local auth state; only the public no-auth read subset below may become executable in this quick task.

## Source Audit

- GOAL: "Make this app chipotle T1-ready" is covered by adding handler-backed T1a coverage for the safe public Chipotle read subset and refreshing readiness/search surfaces.
- STATE invariants: keep the two-tool capability surface, MV3 Wall 1, Wall 2 origin pin, sensitive-origin handling, and fail-closed write posture. No per-app MCP tool, OpenTabs runtime code, direct fetch, or cross-origin credential bridge is introduced.
- Phase 47: Pattern-D/GAPI execution is rejected pending explicit bridge approval. The Chipotle plan must use a narrow no-auth public CORS read proof, not a generic Pattern-D bridge.
- Phase 44/51: Chipotle currently has 16 discovery-pending descriptors. This plan promotes only source-reviewable public reads; other rows stay non-executable.

## T1-ready slugs

- `chipotle.get_ordering_status`
- `chipotle.get_restaurant`
- `chipotle.get_menu`
- `chipotle.get_preconfigured_meals`

## Excluded rows

Do not register these rows in `FsbHandlerChipotle`: `chipotle.get_current_user`, `chipotle.get_extras_campaigns`, `chipotle.get_favorites`, `chipotle.get_last_restaurant`, `chipotle.get_loyalty_points`, `chipotle.get_menu_groups`, `chipotle.get_payment_methods`, `chipotle.get_promotions`, `chipotle.get_recent_orders`, `chipotle.get_reward_categories`, `chipotle.get_rewards`, and `chipotle.find_restaurants`.

`chipotle.find_restaurants` is POST-shaped and currently classified as `write`; it needs live mutation/body review or a separate classification decision before activation. `chipotle.get_menu_groups` is local Vuex state, not a network read, and must not be faked with service-worker localStorage access.

### 1. Add the Chipotle public read handler

Files:
- `catalog/handlers/chipotle.js`
- `extension/catalog/handlers/chipotle.js`
- `tests/capability-head-handlers.test.js`

Action:
- Create a dual IIFE/CommonJS handler module exporting `FsbHandlerChipotle`.
- Use `CHIPOTLE_ORIGIN = "https://www.chipotle.com"`, `CHIPOTLE_SERVICE = "chipotle.com"`, and `SERVICES_BASE = "https://services.chipotle.com"`.
- Register only the four T1-ready slugs listed above as `tier: "T1a"`, `origin: CHIPOTLE_ORIGIN`, `sideEffectClass: "read"`, with params copied from the existing descriptor JSON.
- Build specs only through `ctx.executeBoundSpec`; never call direct `fetch`, `XMLHttpRequest`, `chrome.scripting`, `chrome.tabs`, `chrome.cookies`, or `chrome.webRequest`.
- Use credential-free public CORS specs: `authStrategy: "none"`, `credentials: "omit"`, `origin: CHIPOTLE_ORIGIN`, `extract: "@"`, and JSON accept headers. Do not read or send JWTs, cookies, CSRF tokens, localStorage, sessionStorage, `cmg-vuex`, Authorization, or Bearer headers.
- Use the static public subscription key only if retained from `vendor/opentabs-snapshot/plugins/chipotle/src/chipotle-api.ts`; treat it as a public front-end header, never as a secret, and do not log it.
- Implement these endpoint mappings:
  - `get_ordering_status`: GET `https://services.chipotle.com/onlineorderingstatus?country={country || "US"}`
  - `get_restaurant`: GET `https://services.chipotle.com/restaurant/v3/restaurant/{restaurant_id}?embed=addresses,realHours,onlineOrdering,chipotlane,sustainability`
  - `get_menu`: GET `https://services.chipotle.com/menuinnovation/v1/restaurants/{restaurant_id}/onlinemenu?channelId=web&includeUnavailableItems=true`
  - `get_preconfigured_meals`: GET `https://services.chipotle.com/menuinnovation/v1/restaurants/{restaurant_id}/onlinemeals?includeUnavailableItems=true`
- Map response shapes using the vendored mapper intent in `vendor/opentabs-snapshot/plugins/chipotle/src/tools/schemas.ts`: restaurant/address/hour, menu item, preconfigured meal, and ordering-status booleans.
- Return dual-field `RECIPE_DOM_FALLBACK_PENDING` with `fellBackToDom: true` on missing `ctx.executeBoundSpec`, redirects, 401/403, HTTP errors, non-object/non-array shapes, or missing required response fields.
- Copy `catalog/handlers/chipotle.js` byte-for-byte to `extension/catalog/handlers/chipotle.js`.
- Extend `tests/capability-head-handlers.test.js` with Chipotle fixtures and assertions: four exported read slugs, excluded rows absent, specs use `services.chipotle.com` URL with `origin: https://www.chipotle.com`, `authStrategy: none`, `credentials: omit`, mapped outputs, shape fallback, source banned-token checks, and extension-copy parity.

Verify:

- `node -c extension/catalog/handlers/chipotle.js`
