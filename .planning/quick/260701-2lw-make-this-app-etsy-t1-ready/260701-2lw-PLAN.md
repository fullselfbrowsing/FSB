# Quick Task 260701-2lw: Make Etsy T1-ready

## Goal

Promote Etsy's technically provable same-origin read rows to T1a and make the cart/payment rows terminal as guarded fail-closed, without activating any cart or checkout mutation.

## T1-ready read slugs

- `etsy.search_listings`
- `etsy.get_listing`
- `etsy.list_orders`

## Guarded fail-closed write slugs

- `etsy.add_to_cart`
- `etsy.checkout`

## Tasks

### 1. Add Etsy handler and descriptor backing

Files:
- `catalog/handlers/etsy.js`
- `extension/catalog/handlers/etsy.js`
- `catalog/descriptors/opentabs__etsy__*.json`
- `tests/capability-head-handlers.test.js`

Action:
- Create `FsbHandlerEtsy` as a dual IIFE/CommonJS handler module.
- Implement the three read rows through `ctx.executeBoundSpec` against `https://www.etsy.com/v1/listings/search`, `/v1/listings/{listing_id}`, and `/v1/orders`.
- Return mapped listing/order data only after strict response-shape guards.
- Register `add_to_cart` and `checkout` as guarded fail-closed handlers that return dual-field `RECIPE_DOM_FALLBACK_PENDING` and never call `executeBoundSpec`.
- Mark all five Etsy descriptors as `backing: "handler"`.
