# Implement Steam to be T1 Ready Using GSD Quick Skill

## Scope

Promote Steam from blocked DOM-discovery rows to a conservative bundled T1 surface:

- Safe Steam Store reads resolve to `T1a` handlers pinned to `https://store.steampowered.com`.
- Steam mutation-capable rows stay guarded fail-closed until live mutation-body UAT records method, path, body shape, auth/CSRF carrier, consent, and redaction proof.
- Runtime loading, readiness reporting, origin classification, handler hygiene, guarded-write evidence, and focused tests recognize the Steam surface.

## Tasks

1. Add `catalog/handlers/steam.js` and byte-matching `extension/catalog/handlers/steam.js`.
2. Wire Steam into the T1 manifests and evidence gates: `extension/background.js`, `extension/utils/capability-catalog.js`, `scripts/report-t1-readiness.mjs`, `scripts/verify-t1-port-contract.mjs`, `scripts/verify-origin-classification.mjs`, `scripts/verify-recipe-path-guard.mjs`, and `scripts/coverage-report.mjs`. Leave `catalog/write-activation-evidence.json` unchanged while Steam remains denylisted, because denied Steam rows are blocked rather than active guarded writes.
3. Extend focused tests for handler behavior, dom-to-T1a upgrade, guarded fail-closed behavior, readiness, origin classification, and handler cap/path parity.
