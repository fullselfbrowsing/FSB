## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP bridge message -> extension dispatcher | Trigger bridge messages cross into route handlers and must stay bounded and ownership-aware. |
| extension dispatcher -> background trigger handlers | Route code delegates to background-owned trigger runtime without implementing watcher logic itself. |
| autopilot tool call -> tool executor | Provider-formatted tool calls become background trigger requests through the executor. |
| test gate -> release confidence | Full test/build/smoke commands sample registry, MCP, dispatcher, and autopilot paths together. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-01 | Denial of Service | MCP route/message handlers | mitigate | Routes call bounded background dispatch; companions are already direct/bypass-safe from Plan 18-03. |
| T-18-02 | Information Disclosure / Elevation of Privilege | dispatcher context forwarding | mitigate | Forward FSB-minted `agentId`/`ownershipToken` from bridge payload into background ownership checks; dispatcher does not synthesize status from unscoped data. |
| T-18-04 | Denial of Service | stop route to cleanup path | mitigate | Route delegates to `fsbTriggerHandleToolStop`, whose cleanup order is tested in Plan 18-02. |
| T-18-05 | Information Integrity | status/list routes | mitigate | Route delegates to storage-backed background projection; no alarm-name or SW heap synthesis in dispatcher/executor. |
| T-18-06 | Information Integrity | autopilot/provider route parity | mitigate | Provider visibility tests plus executor source tests prove autopilot uses the same trigger names and background helper as MCP. |
</threat_model>
