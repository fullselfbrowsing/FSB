```ts
type AdapterDoctorRow = Readonly<{

}>;

type BridgeAuthDoctorMetadata = Readonly<{

  secretRotatedAt: number | null;
  secretRotationAgeMs: number | null;
}>;
```

<threat_model>
Boundary: local filesystem/process detection and private bridge-auth state are converted into user-visible doctor output.

- Authority: production adapter registry/detectors and the canonical compatibility module own adapter facts; `readBridgeAuthState` owns private auth state; doctor only projects them. Inject registry/detector/auth-reader/clock dependencies in tests without adding a second policy engine.
- Exact keys and bounds: adapter rows use exactly the eight keys shown above; `compatibilityReason` accepts exactly the canonical `CompatibilityReason` union from Plan 62-01; auth metadata uses exactly `sharedSecretPresent`, `secretRotatedAt`, and `secretRotationAgeMs`. Bound rows to 16, adapter IDs/labels/profile/versions to 64 characters, and local resolved paths to the existing detector/path bound (never more than 4096 characters). Timestamps are finite, non-negative safe integers or null.
- Malformed/prototype failure: accept only own data on plain objects from injected seams; reject accessors/inherited/prototype keys. Detector/auth-reader exceptions become deterministic unavailable/unsupported facts without crashing or claiming health.
- Forbidden data: doctor auth output must never contain the shared secret, previous secret, session secret, fingerprint, bearer/token material, environment values, task/prompt/provider payloads, or private runtime objects. Binary real paths are allowed only in local doctor output and must never feed the browser-safe projection.
- Idempotence: with an injected clock and identical detector/auth inputs, collection and formatting are deterministic and side-effect free. Doctor may probe existing local status but must not start a daemon, connect a provider, rotate a secret, alter storage, or change selection.
- Negative tests: offline bridge/extension, missing binary, malformed version, detector throw, auth-reader throw, missing rotation timestamp, future/malformed timestamp, poisoned/prototype-bearing injected values, and sentinel-secret non-leakage in both text and JSON.
- Security gate: any High/Critical secret leak, unsafe path crossing into browser-facing data, inferred Claude auth, state mutation, or fail-open compatibility result blocks completion.
</threat_model>
