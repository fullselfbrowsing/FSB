<interfaces>
The pure UI model is a closed mapping from background-validated compatibility only:

```js
{
  label: 'Supported' | 'Degraded' | 'Unsupported',
  icon: 'fa-circle-check' | 'fa-triangle-exclamation' | 'fa-circle-xmark',
  className: 'compatibility-badge--supported' | 'compatibility-badge--degraded' | 'compatibility-badge--unsupported',
  detail: string,
  checkedText: string | null
}
```

Exact detail strings are normative:

- Supported: `This CLI is within FSB's fixture-tested compatibility range.`
- Degraded/newer: `This CLI is newer than FSB's fixture-tested range. You can keep it selected; existing start checks still apply.`
- Degraded/stale: `Compatibility evidence is stale. Refresh status to check again.`
- Unsupported: `FSB cannot verify compatibility for this CLI. Refresh status or review setup before starting a task.`
- Claude auth help: `Claude Code does not report an auth state that FSB can safely read.`
</interfaces>

<threat_model>
Boundary: background-projected persisted data becomes visible DOM, accessible descriptions, announcements, and styling in the privileged extension page.

- Authority: background is the sole transport/validator/freshness/storage owner. `providers-panel.js` maps the closed tokens; `options.js` renders. UI may not read raw compatibility storage, inspect a binary, compare/parse a version, invoke doctor, or contact the daemon method directly.
- Exact keys and bounds: accept only the projected compatibility subobject, closed statuses/reasons, a bounded adapter ID, and a validated finite timestamp. Render one group per three agent rows (Claude Code, OpenCode, Codex), zero for seven API rows, and exactly one of three labels/icons/classes; no unknown/loading/unverified fourth badge.
- Malformed/prototype failure: absent, corrupt, accessor, inherited, oversized, future/malformed, unshipped, or unknown values map to Unsupported; `evidence_stale` maps to Degraded. Never copy arbitrary input into `innerHTML`, class names, labels, details, aria text, or announcements.
- Forbidden data/actions: UI compatibility paths must never consume/render binary paths, detected versions, profile internals, sessions, tokens, prompts/provider output, environment values, auth payloads, or secrets. They must not call selection setters, recommendation mutation, preflight/start bypass, storage writes, dirty-state setters, doctor, shell/process/native messaging, or wake code.
- Idempotence/focus: repeated rendering produces the same three-state DOM and does not duplicate descriptions/groups. Refresh preserves row order, checked/focused control, provider kind/id, model/key/endpoint inputs, auth/billing, recommendation, dirty flag, and storage-write count byte-for-byte.
- Negative tests: hostile/prototype inputs, arbitrary version strings, all reason families, stale/missing/corrupt/unshipped evidence, duplicate renders, API rows, refresh success/failure/timeout/fan-out, status transitions, forced colors, dark mode, narrow overflow, and reduced motion.
- Security gate: unresolved High/Critical DOM injection, secret/path/version leak, selection/recommendation/form mutation, focus theft, unsupported-as-supported rendering, or native/shell authority blocks completion.
</threat_model>
