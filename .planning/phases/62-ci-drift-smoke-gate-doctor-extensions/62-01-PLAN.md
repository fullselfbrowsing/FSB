```ts
type CompatibilityStatus = 'supported' | 'degraded' | 'unsupported';
type CompatibilityReason =
  | 'within_tested_range'
  | 'newer_than_tested_range'
  | 'evidence_stale'
  | 'binary_not_found'
  | 'version_missing'
  | 'version_malformed'
  | 'below_minimum'
  | 'wrong_major'
  | 'adapter_unshipped'
  | 'matrix_invalid';

type AdapterCompatibilityContract = Readonly<{
  adapterId: 'claude-code';
  displayLabel: 'Claude Code';
  profileVersion: string;
  minimumVersion: string;
  testedThroughVersion: string;
  supportedMajor: number;
  fixtureManifest: string;
  requiredInitFields: readonly string[];
  requiredResultFields: readonly string[];
  expectedNormalizedSequence: readonly string[];
}>;

type SafeCompatibilitySnapshot = Readonly<{
  schemaVersion: 1;
  checkedAt: number;
  adapters: readonly Readonly<{
    adapterId: string;
    displayLabel: string;
    status: CompatibilityStatus;
    reason: CompatibilityReason;
  }>[];
}>;
```

<threat_model>
Boundary: committed compatibility metadata and fixture JSONL enter daemon policy/classification and the production parser.

- Authority: only `mcp/src/agent-providers/compatibility.ts` may own shipped adapter version bounds, profile/schema version, fixture reference, and classification semantics. Detector, profile, tests, CI, doctor, and extension projections consume it.
- Exact keys and bounds: matrix top level is exactly `{schemaVersion, adapters}` with `schemaVersion: 1`; the Claude row has exactly the contract keys shown above. Reject unknown/missing keys, sparse arrays, more than 16 adapters, field arrays over 32 entries, event arrays over 64 entries, IDs/labels over 64 characters, versions over 32 characters, and fixture references over 256 characters. Versions use strict numeric `major.minor.patch` syntax with no coercion.
- Malformed/prototype failure: validate own enumerable data properties on plain objects only; reject accessors, inherited keys, `__proto__`, `constructor`, `prototype`, NaN/infinite numbers, and malformed/sparse arrays. Invalid metadata or evidence classifies unsupported and never silently widens a range.
- Forbidden data: the matrix/projection must never contain bridge secrets, sessions, tokens, auth payloads, prompts, provider output, local binary paths, home-directory paths, environment values, or captured live payloads. The sole allowed path-like value is a bounded repository-relative fixture-manifest reference.
- Idempotence: deep-freezing is deterministic; classification and safe projection are pure for identical matrix/evidence/clock inputs; repeated parsing never mutates the matrix, fixture, registry, or provider state.
- Negative tests: cover unknown keys, prototype pollution/accessors, malformed and oversized metadata/input, missing init/result, unknown required subtype, duplicate/missing result, below-minimum, newer same-major, wrong-major, and an adapter/fixture/profile roster mismatch.
- Security gate: any unresolved High/Critical gap in validation, prototype handling, secret exclusion, parser bounds, or policy-authority duplication blocks completion.
</threat_model>
