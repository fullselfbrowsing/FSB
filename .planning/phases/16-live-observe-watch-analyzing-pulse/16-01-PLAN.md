<objective>
Create the net-new isolated-world content module `extension/content/trigger-observe.js`: a single-element debounced MutationObserver that reads-and-reports raw `{ text, attributes? }` value deltas to the service worker. This is the core of WATCH-01 (observe the narrowest node, fire with no reload, content reports / SW decides) and the content-side half of WATCH-05 (per-batch selector re-resolution against a stable container + BF-cache-safe teardown discipline + the mandatory observer-leak guard).

Purpose: FSB's genuine moat. The observer survives SW eviction; a low-frequency SW watchdog (Plan 04) re-issues it after eviction/undetected nav. This module writes almost no new logic — it CLONES the shipping `dom-stream.js` MutationObserver skeleton and SWAPS `requestAnimationFrame`→trailing-`setTimeout` (D-06) and `document.body`→a stable container (D-05/D-11/Pitfall 4).
Output: `extension/content/trigger-observe.js`, `tests/trigger-observe.test.js`, and the two new test files appended to the `package.json` test chain.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| page DOM → content script (isolated world) | Untrusted page text is read off the watched element; the observer runs in the isolated world so a page script cannot tamper with it (V1, STACK.md). |
| content script → service worker | The value report crosses to the SW via `chrome.runtime.sendMessage`; the SW-side guard + validation live in Plan 04. This module must emit only the typed `{ text, attributes? }` scalar, never DOM nodes or HTML. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-01-DoS | Denial of Service | MutationObserver on a busy ticker | mitigate | Observe the NARROWEST stable container via `stableAncestor` (never `document.body`/never the leaf); match options to extract kind (`attributeFilter` for attribute); coalesce a burst with the trailing-`setTimeout` debounce → read the value ONCE per batch, never per record (D-05/D-06/D-11, Pitfall 4). |
| T-16-01-TAMP | Tampering | reported value payload | mitigate | `readValue` returns only the extracted scalar string trimmed (`{ text, attributes? }`); never serialize DOM/innerHTML into the report. The page cannot inject structure across the boundary. ReDoS/size caps are enforced downstream in FsbTriggerManager (shipped Phase 15) — this module never compiles or matches a regex (Anti-Pattern 2). |
| T-16-01-LEAK | Denial of Service | observer registry across nav/re-inject | mitigate | Disconnect-all on every real teardown path (beforeunload, non-persisted pagehide, re-inject); idempotent start (disconnect prior before re-observe); the mandatory leak test proves disconnect count === observe count (D-11, Pitfall 2/4). |
