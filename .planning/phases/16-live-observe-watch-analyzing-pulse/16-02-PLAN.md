Add the gentle "analyzing" pulse and the "watching a trigger" label by extending two shipping files additively: `extension/content/visual-feedback.js` (a new `@keyframes fsb-trigger-pulse` variant + `showPulse()`/`clearPulse()` on the Shadow-DOM `ActionGlowOverlay`, reduced-motion handling, and pulse teardown) and `extension/utils/overlay-state.js` (an additive `mode:'trigger-watch'` field + label). Covers VIS-01 (Shadow-DOM glow variant distinct from `run_task`), VIS-02 ("watching" label), VIS-03 (clear-on-teardown half), VIS-04 (reduced-motion).

Purpose: VIS-01 requires a Shadow-DOM glow variant — NOT the inline-style `HighlightManager` (D-07, user-confirmed). Inline styles on a live SPA node get wiped by re-renders AND mutate the page element's own `style`, which the Plan 01 observer could itself see → a self-trigger feedback loop (Pitfall 3). The closed Shadow-DOM overlay (separate from page DOM, tracks the element) avoids both. The pulse animates GPU-composited `opacity`/`transform` only so it does not jank the very ticker it watches (D-08).
Output: the pulse variant + label, plus `tests/trigger-observe-pulse.test.js` (new) and an extension to `tests/test-overlay-state.js`.
</objective>
