Recommended helper shape from `20-RESEARCH.md`:
```javascript
function fsbTriggerCollectDueRefreshPollSnapshots(records, tabId, nowMs, requiredTriggerId) {
  return Object.values(records).filter((snapshot) => {
    if (!fsbTriggerIsRefreshPollSnapshot(snapshot)) return false;
    if (snapshot.target_tab_id !== tabId) return false;
    if (snapshot.trigger_id === requiredTriggerId) return true;
    return Number(snapshot.next_poll_at || 0) <= nowMs;
  });
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| chrome.alarms -> background refresh-poll handler | Alarm events can arrive close together for several triggers on the same tab. |
| trigger registry -> tab reload side effect | Persisted snapshots determine whether a real browser tab is reloaded. |
| content script read -> lifecycle evaluation | Post-reload DOM values cross from page context into service-worker trigger evaluation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-03-01 | Denial of Service | `fsbTriggerHandleRefreshPollAlarm` | mitigate | Use `fsbTriggerRefreshPollTabLocks` and a due batch so same-tab co-located triggers perform one reload per cadence. |
| T-20-03-02 | Elevation of Privilege | refresh-poll ownership validation | mitigate | Validate ownership per trigger before reload/evaluation and mark failures as attention without evaluating them. |
| T-20-03-03 | Tampering | post-reload snapshot state | mitigate | Re-read each latest snapshot before evaluation and skip terminal or non-refresh-poll snapshots. |
| T-20-03-04 | Denial of Service | blocked/challenge pages | mitigate | Preserve restricted/blocked checks before staging values; do not evaluate challenge text. |
</threat_model>
