<objective>
Prove the only safe Windows bootstrap and durable package-owned runtime shape before any native registration or wake implementation exists.

Purpose: NATIVE-01 cannot be complete if Chrome is pointed at a transient npm/worktree path or a Windows command-interpreter shim. This plan resolves the phase's largest packaging uncertainty first.
Output: Workspace-preserving MCP build guard, release-built Windows bootstrap source/build gates, POSIX launcher template, self-contained bundled runtime/integrity contract, and CI/publish packed-artifact proof.

<scope_rationale>This plan intentionally remains one atomic prerequisite despite its broad file roster: Task 63-01-01 first makes every later local MCP build safe in the existing dirty workspace, Task 63-01-02 freezes the runtime and exact bundled dependency closure consumed by installation, and Task 63-01-03 makes the same artifacts mandatory in CI/publish. Splitting any of these three sequential gates would permit a later plan to build over protected output or consume/publish an unproved runtime payload.</scope_rationale>
</objective>
