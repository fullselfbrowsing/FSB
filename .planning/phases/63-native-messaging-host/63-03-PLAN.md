<interfaces>
Frozen request and response:
```typescript
type NativeWakeRequest = Readonly&lt;{
  v: 1;
  action: 'wake';
  correlationId: string;
}&gt;;

type NativeWakeResponse = Readonly&lt;{
  v: 1;
  correlationId: string;
  outcome: 'already_running' | 'started' | 'unavailable' | 'failed';
  reason: NativeWakeReason;
}&gt;;
```

Exact reason vocabulary:
```text
daemon_already_ready | daemon_started_ready | daemon_identity_mismatch |
daemon_protocol_mismatch | runtime_invalid | wake_lock_timeout |
serve_spawn_failed | serve_readiness_timeout | internal_failure
```

Exact outcome/reason compatibility:
```text
already_running -> daemon_already_ready
started         -> daemon_started_ready
unavailable     -> daemon_identity_mismatch | daemon_protocol_mismatch | runtime_invalid
failed          -> wake_lock_timeout | serve_spawn_failed | serve_readiness_timeout | internal_failure
```
</interfaces>
