<task type="auto">
  <name>Task 1: Add the readiness classifier and report generator</name>
  <files>scripts/report-t1-readiness.mjs</files>
  <action>
Create a Node-builtins-only dual-export script modeled on scripts/coverage-report.mjs.

The script should:
- load the committed generated catalog from extension/catalog/recipe-index.generated.js;
- plant globalThis.FsbRecipeIndex before requiring capability-catalog.js;
- require extension/catalog/handlers/github.js, slack.js, notion.js, and gitlab.js best-effort;
- call seedHeadHandlers();
- call resolve(slug, "https://" + service) for every descriptor;
- classify each row into one of:
  - t1-ready
  - t1-guarded-fail-closed
  - learn-pending
  - discovery-pending
  - blocked
  - unknown
- classify guarded fail-closed by known handler slugs whose handler returns `RECIPE_DOM_FALLBACK_PENDING` for writes, using a conservative allowlist derived from existing guarded-write tests or handler metadata;
- derive origin class from the committed denylist/sensitive-origin classifier where possible, falling back to unknown rather than guessing;
- infer auth pattern and same-origin/separate-origin feasibility conservatively from handler metadata, recipe origin, descriptor service, and existing origin-classification helpers;
- assign a next action such as "already executable", "live mutation-body UAT", "same-origin read candidate", "Pattern-D candidate", "GAPI bridge candidate", "learn via network capture", or "keep DOM/discovery".

The CLI should write both:
- .planning/phases/44-t1-readiness-inventory-status-surface/44-T1-READINESS.json
- .planning/phases/44-t1-readiness-inventory-status-surface/44-T1-READINESS.md

It should also print a short totals summary.
  </action>
  <verify>
    <automated>node scripts/report-t1-readiness.mjs</automated>
  </verify>
  <done>The script generates markdown and JSON evidence without throwing and reports every descriptor exactly once.</done>
</task>
