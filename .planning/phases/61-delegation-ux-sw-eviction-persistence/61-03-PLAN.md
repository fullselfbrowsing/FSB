<interfaces>
```js
mcpBridge.addEventObserver(async function(event) { /* observer roster is global; invocation tail is per pending correlation */ })
mcpBridge.retainDelegationHeartbeat(ownerId) // refcount +1; starts one loop
mcpBridge.releaseDelegationHeartbeat(ownerId) // refcount -1; stops at zero
mcpBridge.getDelegationConnectionSnapshot()
// { state: 'connected'|'disconnected', consecutiveMisses, lastAckAt }
```

Each `_extPending` record owns `{ eventTail, observerError }`. A matching `ext:event` appends that request's observers in registration order to only that record's tail; the matching `ext:response` snapshots and awaits that same tail before settling. Failure rejects only its correlation, is not inherited by unrelated requests, and cannot be erased by a later nominal response.

Heartbeat frames are ordinary bridge protocol messages `{ type:'mcp:ping', ts, nonce }` with a fresh bounded opaque nonce. The daemon replies `{ type:'mcp:pong', ts, nonce }`, echoing the exact validated nonce without interpreting it. Only the matching pong for the currently outstanding nonce acknowledges a beat. Interval is 20,000 ms and three consecutive misses classify `disconnected`; reconnect does not classify restart or restart a task.
</interfaces>
