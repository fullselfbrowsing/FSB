## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Service-worker runtime response → options page | Client evidence may be missing, stale, malformed, or fail in transit. |
| Observed evidence → advisory UI | Status/recommendation could be mistaken for current truth or user consent. |
| Agent definition/auth state → billing copy/link | Financial meaning must reflect confirmed mode and fixed destinations. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-58-01 | Spoofing | evidence renderer | mitigate | Only helper-allowlisted agents receive selectable/status/recommendation treatment; raw identities remain informational. |
| T-58-02 | Tampering | refresh/recommendation | mitigate | Separate render paths, no selection/storage calls, frozen inputs, exhaustive non-mutation tests. |
| T-58-04 | Information Integrity | usage/billing UI | mitigate | Real metrics or em dashes only; conditional subscription label; provider-specific qualification and exact-source tests. |
| T-58-05 | Spoofing | billing links | mitigate | Frozen exact official HTTPS URLs, visible labels, opener isolation, no response-derived URLs. |
| T-58-06 | Denial of Service | runtime/storage refresh | mitigate | Defensive envelope validation, in-flight coalescing, 100 ms debounce, prior snapshot retention, no polling. |
</threat_model>
