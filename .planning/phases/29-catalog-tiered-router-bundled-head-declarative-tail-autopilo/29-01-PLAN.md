Router entry (Plan 02 implements; tests import via require()):
- `FsbCapabilityRouter.invoke(slug, args, { origin, tabId })` -> Promise
  - hit:  `{ success:true, ...result, tier }`  (tier ∈ 'T0'|'T1a'|'T1b')
  - miss: `{ success:false, code, errorCode, error, ... }` where code matches /^RECIPE_.+$/
  - no catalog entry / no recipe -> code 'RECIPE_NOT_FOUND'
  - tier 'T2' (learned stub) -> code 'RECIPE_LEARN_PENDING'
  - tier 'T3' (DOM fallback seam) -> code 'RECIPE_DOM_FALLBACK_PENDING' AND no executeTool/chrome.scripting call

Catalog entry (Plan 02 implements):
- `FsbCapabilityCatalog.resolve(slug, origin)` -> `{ tier, handler?, recipe?, descriptor? } | null`
  - a slug is EITHER T1a OR T1b (no runtime tie-break); origin biases search ranking only, never the tier choice for a known slug (RESEARCH Open Q3)

Handler entry (Plan 03 implements):
- per-service module exporting a slug-keyed object: `{ '<slug>': { tier:'T1a', origin, sideEffectClass, async handle(args, ctx) } }`
  - `ctx = { origin, tabId, executeBoundSpec, interpretRecipe }`
  - handler builds bound spec(s) and calls `ctx.executeBoundSpec(spec, tabId)` — NEVER chrome.scripting itself

Autopilot branch (Plan 05 implements):
- a pre-`executeTool` guard in tool-executor.js: `if (CAPABILITY_TOOL_NAMES.has(name)) return executeCapabilityToolForAutopilot(name, params, tabId)` where `CAPABILITY_TOOL_NAMES = new Set(['invoke_capability','search_capabilities'])`; calls `globalThis.FsbCapabilityRouter.invoke(...)`

Execution primitive (already shipped — capability-fetch.js:272):
- `FsbCapabilityFetch.executeBoundSpec(spec, tabId)` -> `{ success, status, finalUrl, redirected, data, text }` | `{success:false, code:'RECIPE_ORIGIN_MISMATCH', ...}` (re-pins active tab BEFORE any side effect)
