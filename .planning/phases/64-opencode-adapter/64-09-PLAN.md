<objective>
Generalize the extension's authorization path around one closed canonical Claude/OpenCode provider model.

Purpose: The side panel may request a task, but only background-owned saved settings, compatibility/preflight, and provider-bound consent may select the daemon adapter.
Output: Canonical provider metadata, exact two-provider preflight, isolated consent/trust, and immutable accepted-run routing context.

<scope_rationale>The provider table, preflight, consent, and background start path are one authorization chain. Generalizing only one consumer would permit identity/billing drift or let a stale challenge authorize a different adapter.</scope_rationale>
</objective>
