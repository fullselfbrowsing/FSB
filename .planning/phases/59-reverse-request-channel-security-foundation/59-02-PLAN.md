<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser HTTP upgrade → localhost bridge | Any webpage/extension can dial localhost; socket creation must not imply trust. |
| Host header → loopback authority | DNS rebinding can deliver attacker-controlled Host while TCP terminates at 127.0.0.1. |
| Pair command/file → extension session | Raw credential crosses only by explicit user copy and subprotocol header. |
| `serve` restart → existing extension credential | Rotation must revoke stale credentials without losing durable exact Origin. |

## Blocking Threat Register

| Threat | Severity | Mitigation | Blocking evidence |
|--------|----------|------------|-------------------|
| T59-01 CSWSH | Critical | exact parsed extension Origin at HTTP upgrade before handler | evil web Origin rejection + zero handler count |
| T59-02 DNS rebinding | Critical | exact loopback Host and port before handleUpgrade | `Host: evil.com:<port>` rejection despite loopback target |
| T59-03 malicious extension/stale credential | High | durable exact Origin, 32-byte rotated token, sessionId bound to socket and revalidated per ext frame, explicit reset | wrong Origin/token/no-token, active-socket rotation, reset/new-ID rebind cases |
| T59-04 credential disclosure | High | 0600 atomic file, explicit stdout only, header-only auth, no shared diagnostics | pair/status/doctor/log substring assertions |
| T59-08 remote bind | Critical | closed bind allowlist before listen | 0.0.0.0/LAN/`::` pre-listen rejection |

All High/Critical threats have automated real-socket/filesystem evidence. Any pre-handler counter increment on a hostile fixture blocks the plan.
</threat_model>
