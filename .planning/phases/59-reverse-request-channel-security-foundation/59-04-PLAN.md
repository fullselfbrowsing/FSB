Client API:

```js
mcpBridgeClient.sendExtRequest(method, payload, {
  timeout: 30000,
  onEvent: (eventName, eventPayload) => {}
}); // Promise<Record<string, unknown>>
```

| Boundary | Description |
|----------|-------------|
| Options page → service worker | Both are trusted extension contexts, but a runtime message must not transport the raw credential. |
| Session storage/memory → WebSocket constructor | Credential may appear only as offered subprotocol, never state/log/URL/frame. |
| Reverse frames → client pending map | Events/finals may be malformed, duplicated, late, spoofed, or arrive after reconnect. |
| User pairing input → provider settings/UI | Secret transport state must not contaminate durable BYOK/provider configuration or displayed copy. |

| T59-03 stale/malicious credential | High | exact regex, trusted session storage, reconnect-only elevation, authenticated probe, paired/expired distinction | malformed/old/active-rotation/probe/close/reload lifecycle fixtures |
| T59-04 secret disclosure | High | no URL/frame/runtime payload, session-only record, blank input, no state/log output | serialized storage/state/log/source absence assertions |
| T59-05 duplicate request after reconnect | High | independent pending map, settle once, close rejection, no replay | event/final/duplicate/timeout/close/reconnect fixtures |
| T59-06 direction confusion | High | intercept ext responses/events before inbound MCP dispatcher | background/client routing tests with invocation counts |

All High issues are blocking automated tests. Live rendering/pairing observation remains deferred UAT but cannot substitute for any source/lifecycle gate.
