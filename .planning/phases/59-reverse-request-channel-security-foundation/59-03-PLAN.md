<interfaces>
Extend `BridgeOptions` additively:

```ts
export type ExtRequestHandler = (
  request: ExtRequest,
  emit: (event: ExtEvent) => void,
) => Promise<Record<string, unknown>>;

export interface BridgeOptions {
  // existing fields unchanged
  capabilities?: BridgeCapability[];
  handleExtRequest?: ExtRequestHandler;
}
```

Only a bridge configured with both `capabilities: ['agent-spawn']` and `handleExtRequest` is locally capable. Phase 59 production construction passes neither; topology tests inject them. Stable terminal errors use `makeExtError` codes:

- `agent_provider_offline`, retryable true — no current handler/relay
- `bridge_topology_changed`, retryable true — selected target disconnected before final response
- `invalid_ext_request`, retryable false — malformed/duplicate active ID/handler invalid result
- `ext_unauthorized`, retryable false — upgrade lacked authority

`method: 'bridge.auth-status'` is the sole built-in transport method. After current sessionId/Origin revalidation it returns `{ authorized: true }` locally without consulting capabilities, handlers, or relays. It exists only so pairing can distinguish an open legacy socket from an authenticated reverse socket.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Authorized extension socket → hub router | Valid authentication does not make arbitrary frame shapes/methods safe. |
| Hub → capable relay | Capability advertisement is authority selection and must be explicit, closed, and connection-scoped. |
| Relay response/event → extension | A relay can spoof another route, duplicate finals, or send late events. |
| Topology churn → pending request | Automatic failover/replay could duplicate a future process spawn. |

## Blocking Threat Register

| Threat | Severity | Mitigation | Blocking evidence |
|--------|----------|------------|-------------------|
| T59-05 confused deputy/misroute | High | closed capability registry, local-first/first-relay deterministic selection, selected-relay response check | local/first/offline/spoof route cases |
| T59-05 duplicate execution on churn | High | independent state, settle/delete once, terminal topology error, no replay | relay-mid-frame and hub-exit no-replay cases |
| T59-06 legacy wire drift | High | optional omission and unchanged MCP correlation paths | exact default frame + version parity + old topology cases |
| Unauthorized reverse route | Critical | upgrade auth plus current sessionId/Origin revalidation on each ext frame and strict parser before route selection | unauthorized/malformed/active-rotation handler and relay invocation counts remain zero |

All High/Critical issues block. No test may treat reconnection or retry as a successful replacement for deterministic settlement.
</threat_model>
