|----------|-------------|
| chrome.storage.session → handler/restore | A persisted snapshot re-enters SW code as untrusted on every alarm tick / cold boot (may be malformed/poisoned after a code reload). |
| chrome.alarms (persisted across SW lifetime) → onAlarm/restore | A stale `fsbTrigger:*` alarm may fire (or be enumerated by getAll()) with no backing snapshot after a code reload. |
| Tab lifecycle (chrome.tabs.onRemoved) → handleTriggerTabRemoved | An arbitrary `tabId` crosses in; must be validated before scanning. |
