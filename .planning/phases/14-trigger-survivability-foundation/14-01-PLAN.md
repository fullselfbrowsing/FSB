<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SW code → chrome.storage.session | Persisted snapshot crosses into a Chrome storage area; on a later read it re-enters SW code as untrusted (may be malformed/poisoned after a code reload or version change). |
| Caller (Plan 14-02 lifecycle / future Phase 15 manager) → trigger-store API | `trigger_id` / `snapshot` arguments are untrusted at the store boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01 | Denial of Service | `_readEnvelope` reading a poisoned/version-mismatched payload | mitigate | Return the canonical empty `{v:1,records:{}}` on missing key / version mismatch / malformed / any error — never null, never throw (cloned verbatim from mcp-task-store.js :70-91). Verified by the version_mismatch_returns_empty test case. |
| T-14-02 | Denial of Service | `writeSnapshot`/`readSnapshot`/`deleteSnapshot` with bad `trigger_id`/`snapshot` | mitigate | V5 input validation: silently no-op on `!string` id / `!object` snapshot (cloned from :128-130); never throw out of the store. |
| T-14-03 | Denial of Service | Any storage call when `chrome` is unavailable or `set/remove` throws | mitigate | Best-effort no-throw posture: lazy `_getChrome()` returns null safely; every storage call wrapped in try/catch (:88-90, :117-119). Verified by chrome_unavailable_no_throw. |
| T-14-04 | Tampering | `agent_id` field (cross-agent ownership) | accept (this phase) | Store `agent_id` faithfully now (V4); do NOT drop or normalize it. Cross-agent ownership ENFORCEMENT is Phase 18 — out of scope here, accepted as a reserved-field-only posture. |
| T-14-SC | Tampering | npm/pip/cargo installs | mitigate | N/A — this plan installs ZERO packages (clone of an in-tree module; `node`+built-in `assert` only). No registry lookup, no postinstall. If a dependency need is discovered it is a scope violation — escalate. |
</threat_model>
