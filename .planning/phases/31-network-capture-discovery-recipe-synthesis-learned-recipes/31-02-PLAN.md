<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| page -> SW capture | The page's real network traffic crosses into the SW via the SW-owned `chrome.debugger` attachment; the page CANNOT inject CDP events (only the SW attachment fires onEvent). |
| raw CDP event -> redacted ObservedCall | The redaction boundary: a raw request/response (headers, body, query) is reduced to shape BEFORE anything leaves the handler frame. |
| capture-start -> consent gate | A capture session is authorized per-origin by the Phase-30 consent gate before any attach. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-02-LEAK | Information Disclosure | redactRequest/redactResponse | mitigate | Header VALUES and bodies are NEVER read (names-only + structural exclusion, D-06/D-07); the query string is dropped (path only); getResponseBody is never called (D-08). The Wave-0 no-secret test is the executable proof. |
| T-31-02-UNCONSENTED | Access Control | startSession | mitigate | The capture-start runs the identical Phase-30 gate (getConsentForOrigin != 'off' AND !isDenied AND sensitive-extra-confirm) before attach; a default-Off / denied origin is never captured (DISC-04/D-03). |
| T-31-02-SPOOF | Spoofing / Tampering | _onCdpEvent | mitigate | Events arrive ONLY from the SW-owned debugger attachment — a malicious page cannot forge them. Defense-in-depth: same-origin filter (cross-origin dropped, D-04) gates which calls are even observed. |
| T-31-02-DETACH | Denial of Service | endSession | mitigate | Ownership-checked release (Network.disable + listener removal; detach only if weAttached and no Input op holds the tab, Pitfall 1) + onDetach teardown — a capture session never breaks the KeyboardEmulator Input emulation and never leaks an attachment (DISC-02). |
| T-31-02-SC | Tampering | npm installs | accept | Zero new packages (RESEARCH audit). |
</threat_model>
