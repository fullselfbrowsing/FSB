<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ObservedCall -> synthesized recipe | A redacted (but still untrusted-shaped) capture becomes a recipe that must pass the closed-vocab schema before it can be promoted/executed. |
| synthesized recipe -> declarative replay coverage | The synthesizer's authStrategy output must stay within what the declarative replay path can execute (the from:'response' cap). |
| per-origin store -> routing | A stored learned recipe is surfaced to the router only for its OWN origin and only when not quarantined. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-03-HOSTILE | Tampering | synthesize | mitigate | The synthesized recipe is an UNTRUSTED payload validated against the closed-vocab `validateRecipe` (forbidden-name pre-scan + additionalProperties:false) BEFORE it can be returned/promoted (D-12); a non-conforming synthesis returns null. |
| T-31-03-UNEXEC | Tampering / DoS | synthesize authStrategy | mitigate | authStrategy is capped to declarative-executable values; `csrf.from:'response'` is NEVER emitted (the GATING cap, D-11) — preventing a schema-valid-but-unreplayable recipe that would fail the promote gate or mis-execute. |
| T-31-03-CONFUSED | Elevation of Privilege | getLearned | mitigate | The store scopes HARD by origin (`recipe.origin === origin`, Pitfall 6) — a learned recipe for origin A is never surfaced for origin B; the executeBoundSpec origin-pin is the downstream backstop. |
| T-31-03-LEAK | Information Disclosure | recipe core + stored envelope | mitigate | The recipe stores request SHAPE only (method, template, header names, csrf-source, extract '@', origin); the synthesizer never reads a body and `extract:'@'` avoids inferring a deep path from a body (D-08); no response body / PII is ever stored (LEARN-02). |
| T-31-03-SC | Tampering | npm installs | accept | Zero new packages (RESEARCH audit). |
</threat_model>
