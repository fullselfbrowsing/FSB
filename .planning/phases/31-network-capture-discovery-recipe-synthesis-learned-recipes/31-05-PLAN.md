<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| learned store -> catalog resolve | The catalog surfaces a learned recipe only for its OWN origin; a cross-origin learned recipe must never be returned. |
| catalog T2 -> router dispatch | A learned T2 recipe is executed through the SAME declarative replay path as bundled recipes, with the consent gate and origin-pin unchanged upstream/downstream. |
| learned recipe -> interpreter | The router threads `trustedProvenance:'local'` so the loader (router) vouches the recipe — the recipe never self-declares (HI-01, Plan 04). |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-05-CONFUSED | Elevation of Privilege | catalog resolve / _getLearned | mitigate | The learned lookup is hard origin-scoped in the store (Plan 03, Pitfall 6) and resolve returns the learned recipe only for the active origin; the executeBoundSpec origin-pin re-asserts the active-tab origin downstream -> RECIPE_ORIGIN_MISMATCH with no fetch on any mismatch. |
| T-31-05-GATEBYPASS | Access Control | router case 'T2' | mitigate | The consent gate (:455-470) runs BEFORE the dispatch switch and is untouched — a learned T2 invoke is gated identically to every other tier; the dispatched outcome is still audited (no secrets). |
| T-31-05-VOUCH | Tampering | _runDeclarativeTier opts thread | mitigate | The router supplies `trustedProvenance:'local'` as the LOADER's vouch (the trusted channel); the recipe payload never asserts it (HI-01). T0/T1b omit opts and stay exempt-by-source — no broadening. |
| T-31-05-DRIFT | Tampering / DoS | addLearnedRecipe snapshot | mitigate | addLearnedRecipe mutates the ONE INDEX_OPTIONS-built index and re-snapshots with a bumped version (Pitfall 5) — preventing a loadJSON options-drift throw on SW wake that would silently drop the learned entry from search. |
| T-31-05-SC | Tampering | npm installs | accept | Zero new packages (RESEARCH audit). |
