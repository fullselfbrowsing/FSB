# 23-03 Plan: Angular Dashboard Renderer Migration

## Goal

Move the Angular dashboard to the same PhantomStream viewer wrapper contract used by the static dashboard, with tests preventing the two surfaces from drifting.

## Tasks

1. Add or reuse the Angular viewer host element and load `assets/phantom-stream-viewer.js` before dashboard message handling.
2. Replace Angular manual iframe `srcdoc` construction with wrapper viewer dispatch for `ext:dom-snapshot`.
3. Replace Angular manual `[data-fsb-nid]` mutation application with wrapper viewer dispatch for `ext:dom-mutations`.
4. Preserve Angular preview states, loading/restricted/error/frozen behavior, side-channel UI, client badge, tooltip diagnostics, layout controls, and remote-control overlay affordances.
5. Add parity/source-contract tests that compare static and Angular use of:
   - shared wrapper global;
   - snapshot dispatch;
   - mutation dispatch;
   - scroll/overlay/dialog forwarding;
   - viewer mapping for remote-control geometry;
   - stale-session and resync behavior.
6. Run the Angular build/smoke checks that are practical in this workspace.

## Verification

```bash
node tests/phantom-stream-dashboard-parity.test.js
node tests/dashboard-runtime-state.test.js
npm --prefix showcase/angular run build
npm run validate:extension
git diff --check
```

## Done When

- Angular and static dashboards consume one shared wrapper contract.
- Angular no longer carries a second generic DOM diff renderer.
- Source-contract tests fail if one surface drifts back to a local renderer implementation.
