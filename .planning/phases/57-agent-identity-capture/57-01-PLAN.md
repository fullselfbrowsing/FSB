<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client → SDK server/AgentScope | Untrusted `initialize.clientInfo` crosses into FSB as observability metadata. |
| Local PATH/filesystem → daemon detector | Executable resolution, version output, and platform config-path existence are external machine state. |
| Daemon → extension bridge | Optional identity/inventory fields cross the existing byte-stable MCP bridge. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T57-01 | Spoofing | `AgentScope.clientInfoSupplier` | mitigate | Carry only optional name/version evidence; never use it for authorization, ownership, or permission checks; Task 1 freezes the null and unknown-field paths. |
| T57-02 | Elevation of Privilege | `client-inventory.ts` Claude probe | mitigate | Use fixed executable candidates, argv `['--version']`, 3 s timeout, 64 KiB cap, and never `shell: true`; Task 2 injects and asserts the exact invocation. |
| T57-06 | Tampering | `MCPMessageType` and `agent:register` payload | mitigate | Add one new type and optional fields only; preserve all existing strings and the exact legacy empty payload; focused source/contract tests plus final full regression detect drift. |
| T57-07 | Denial of Service | CLI version probe | mitigate | Bound execution to 3 s and 64 KiB, memoize once per process, and treat every probe error as `detected:false` instead of blocking runtime startup. |
</threat_model>
