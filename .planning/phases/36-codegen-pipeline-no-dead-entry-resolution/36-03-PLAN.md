<!-- The side-effect inference contract (RESEARCH Mechanic 2, read live across 4 plugins). -->

Class ordering (MAX-merge / fail-safe-high): read < write < destructive.

Derivation priority (Mechanic 2):
  1. GraphQL/RPC carve-out (FIRST): if transport is graphql/gql/gqlRequest/persisted-query, the HTTP
     method is uninformative (always POST) -> classify by OP-NAME VERB; an ambiguous GraphQL op
     fails-safe to write; a GraphQL op is NEVER auto-classed read merely because no apiPost appears.
  2. Named verb helper: apiGet->read; apiPost/apiPut/apiPatch->write; apiDelete->destructive.
  3. Generic api({method}): GET/HEAD->read; POST/PUT/PATCH->write; DELETE->destructive; no literal -> default GET -> read BUT cross-check vs name verb.
  4. Op-name verb prefix (always computed; the cross-check partner):
       list_/get_/search_/read_/fetch_/find_ -> read
       create_/update_/delete_/add_/remove_/set_/void_/archive_/merge_/move_/finalize_ -> write (delete-family -> destructive)
  5. Override table (highest specificity, applied LAST to UPGRADE only — a FLOOR, never a downgrade):
       void_invoice, delete_customer, cancel_subscription, refund_charge, delete_record, archive_project -> destructive
       merge_pull_request -> write

Disagreement rule: take the MAX across all computed signals. Example: void_invoice with
api('/invoices/void', {method:'POST'}) -> method says write, override says destructive -> destructive.
A linear graphql op named archiveIssue -> carve-out + name verb -> write (never read).

The importer (Plan 01) persists signals into each descriptor:
  provenance.signals = { transportHelper, httpMethod, opNameVerb }
so this gate re-derives WITHOUT re-parsing TS.

Existing gate to mirror (shape/dual-export idiom): scripts/verify-classification-gate.mjs
  (export function classifyGate; a CLI that runs only on direct invocation, never on import) +
  scripts/verify-recipe-path-guard.mjs (the CI-gate skeleton: read files, check, process.exit(1) on fail).
This plan REPLACES the Plan-01 stub `scripts/verify-catalog-crosscheck.mjs` (already registered in
package.json validate:extension) — do NOT touch package.json.
