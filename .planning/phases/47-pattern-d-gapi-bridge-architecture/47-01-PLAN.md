# 47-01 Plan: Pattern-D Decision

## Objective

Determine whether separate-origin app APIs can be safely executed through the current T1 substrate.

## Tasks

1. Inspect the existing origin classifier and runtime origin-pin controls.
2. Quantify Pattern-D candidates from the readiness report.
3. Decide whether generic separate-origin execution is implementable in this phase.
4. Record the decision in a durable architecture note.

## Acceptance

- Pattern-D is either implemented with negative controls or explicitly rejected.
- The decision covers Linear, Datadog/Jira, Supabase, and cloud-console style APIs.
