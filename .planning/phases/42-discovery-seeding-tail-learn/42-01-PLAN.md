  <name>Task 1: Build the discovery-seed harvester + emit discovery-seeds.json</name>
  <files>scripts/harvest-discovery-seeds.mjs, extension/config/discovery-seeds.json</files>
  <action>
Create `scripts/harvest-discovery-seeds.mjs` (build-time, ESM, alongside `import-opentabs-catalog.mjs`).
For each app under `vendor/opentabs-snapshot/plugins/<app>/`: derive the bare ORIGIN by reusing the importer's
parse (read `package.json.opentabs.urlPatterns[0]`, `match(/:\/\/([^/]+)\//)`, strip a leading `*.`, prefix
`https://`). Then read the app's `src/*-api.ts` and extract endpoint HINTS — the `{ op, method, path }` triples
derivable from the file text WITHOUT executing it: scan for fetch target URLs / GraphQL endpoint constants
(e.g. `GRAPHQL_ENDPOINT = 'https://client-api.linear.app/graphql'` -> path `/graphql`), `/v1/` style path
literals, and `method:` / default-method literals. Map each exported tool op name (the `*-api.ts` exported
function names) to its path/method when resolvable; when a path is not statically resolvable, emit the op with a
null path (recognition still keys on origin). Emit `extension/config/discovery-seeds.json` keyed by bare origin:
`{ "<origin>": { hints:[{op,method,path}], provenance:{app,source} }, ..., "_meta": { generator, vendorSha,
generatedAt, originCount, hintCount } }`. PROVENANCE-stamp it: `vendorSha` = the vendored snapshot SHA (read it
from `vendor/opentabs-snapshot/.git` HEAD or a vendored VERSION/manifest file if present; if no SHA is
resolvable, stamp `vendorSha:'unpinned'` and emit a single console.warn — never throw). Skip empty-origin
self-hosted apps (no urlPatterns host) with a clean continue (mirror the importer's BLOCKER-C skip), never abort
the batch. Per DSEED-01: the harvester writes METADATA ONLY — it adds NO host permission and the hints are never
executed. Run the harvester to produce the committed `discovery-seeds.json`.
  </action>
  <verify>
    <automated>node scripts/harvest-discovery-seeds.mjs && node -e "const s=require('./extension/config/discovery-seeds.json'); const origins=Object.keys(s).filter(k=>k!=='_meta'); if(origins.length<50)throw new Error('too few origins: '+origins.length); if(!s._meta||!s._meta.vendorSha)throw new Error('missing provenance'); console.log('origins',origins.length,'hints',s._meta.hintCount)"</automated>
  </verify>
  <done>discovery-seeds.json exists, keyed by bare origin, with >=50 origins, a _meta provenance block (generator + vendorSha + counts), and endpoint hints harvested from the vendored *-api.ts. The harvester degrades (warn, never throw) on a missing SHA or an empty-origin app.</done>
