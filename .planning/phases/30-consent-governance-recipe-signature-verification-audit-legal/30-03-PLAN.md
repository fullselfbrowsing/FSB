## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| server/learned recipe → interpretRecipe | A non-bundled recipe is untrusted DATA; it must be signature-verified over a deterministic canonical form before it can bind to a credentialed call. |
| bundled trusted public key → verify | The key is bundled (Wall-1), never remotely fetched; a forged key cannot be injected at runtime. The fixture key is test-only. |
| denylist config → gate | The denylist is a hard, checked-first deny; a denylisted origin must be non-enableable regardless of stored policy. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-30-10 | Tampering | signature malleability via JSON re-serialization | mitigate | RFC-8785 JCS canonicalization (deterministic byte string, UTF-16 key sort, integer-only number rule) before sign/verify; the signed-payload scope excludes the signature field (Pitfall 3). |
