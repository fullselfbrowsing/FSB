# 45-02 Plan: Current-Catalog Contract Verifier

## Goal

Apply the scaffold to the current catalog so future ports cannot silently bypass the contract.

## Scope

- Add a verifier that reads the current readiness report.
- Validate current T1 handler rows for handler presence, origin match, no direct browser scripting, no direct fetch, no dynamic code, and no secret-bearing console logs.
- Validate active write rows against an explicit UAT evidence set.
- Dynamically invoke every readiness-sourced `t1-guarded-fail-closed` row with a recording context and require the recorder to remain empty.
- Verify the MCP tool registry still does not expose app-specific capability tools.

## Files

- `scripts/verify-t1-port-contract.mjs`
- `tests/t1-port-contract-gate.test.js`
- `package.json`

## Verification

- `node scripts/verify-t1-port-contract.mjs`
- `node tests/t1-port-contract-gate.test.js`
- `npm run validate:extension`

## Acceptance Criteria

- Current catalog passes the verifier.
- Negative controls prove mutation-firing guarded writes fail, active writes without UAT evidence fail, app-specific MCP tool names fail, and handler source scans catch direct browser/network bypasses.
- `validate:extension` runs the new verifier.
