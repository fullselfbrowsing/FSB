# 45-01 Plan: Port Contract Library + Scaffold CLI

## Goal

Create the reusable Phase 45 scaffold for same-origin reads, same-origin writes, guarded writes, and separate-origin candidates.

## Scope

- Add a library under `scripts/lib/` that validates port proof metadata.
- Add a CLI that prints a markdown checklist for a descriptor slug and optional port type.
- Cover the validator and renderer with focused tests.

## Files

- `scripts/lib/t1-port-contract.mjs`
- `scripts/scaffold-t1-port.mjs`
- `tests/t1-port-contract.test.js`

## Verification

- `node tests/t1-port-contract.test.js`

## Acceptance Criteria

- The validator rejects missing origin pin, missing `executeBoundSpec` proof, missing body-shape fallback, secret-logging risks, active writes without UAT evidence, and executable separate-origin candidates.
- The CLI renders checklists for same-origin reads, same-origin writes, guarded writes, and separate-origin candidates.
- The checklist calls out origin pin, `executeBoundSpec` only, logged-out/body shape guard, no secret logs, consent/audit compatibility, router parity, fallback byte stability, and write UAT/fail-closed status.
